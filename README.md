## 这是什么

**典雅白话小说引擎**是一套运行在 Claude Code 上的 AI 写作辅助系统。它由 **7 枚协作 skill** 组成，覆盖从一句话灵感到成品小说的完整创作流程。

与"让 AI 帮你写"不同，这套系统是一套**约束系统**——它为 Claude 建立了一套完整的写作框架：

- 写什么风格？→ **典雅白话**（四个否定之后剩下的那个东西）
- 怎么写？→ **三层风格规则**（句子→段落→篇章）
- 写完怎么检查？→ **两道硬性门禁**（字数→AI味）
- 武侠怎么写？→ **专门的武侠引擎**（招式命名/打斗节奏/江湖道义）
- 怎么让文字像人写的？→ **去AI三遍法 + 35项指纹扫描**

**核心理念**：最好的中文小说，读起来既不像翻译腔，也不像 AI 腔——它像一个人在对你娓娓道来，每个字都恰到好处。

## 核心理念：典雅白话

典雅白话不是文言文，不是大白话，不是翻译体，不是公文腔。它是**四个否定之后剩下的那个东西**：

| 排除项 | 特征 | 反面例子 |
|--------|------|----------|
| 文言腔 | 滥用"之乎者也"、刻意古雅 | "然则今日之事，岂非天命乎？" |
| 翻译体 | 欧化长句、被动语态、"作为"开头 | "作为一位资深教师，他被他学生的行为深深地感动了。" |
| 公文腔 | 空洞抽象、套话连篇 | "进一步提升综合素养，切实增强责任意识" |
| AI腔 | 过度工整、万能搭配、解释充分 | "不是愤怒，而是深深的失望。他深吸一口气，眼中闪过一丝复杂的情绪。" |

**典雅白话 = 精准 + 自然 + 有节奏。** 每个词都有理由存在，每个句子都有呼吸，每段话都有起伏。

它在这样一个光谱上：

```
文言文 ─── 古风典雅白话 ─── 普通白话 ─── 口语化 ─── 网络用语
          ↑ 本引擎的目标
```

### 调用规则

1. **主引擎先行**：任何小说写作都必须先加载 `dianya-novel-engine`
2. **按篇幅分派**：主引擎 Phase 1 判断篇幅后，自动加载对应子引擎
3. **按题材叠加**：武侠/江湖题材额外加载 `dianya-wuxia`
4. **交叉校验强制**：每章写完后，子引擎强制调用 `dianya-cross-check`
5. **去AI不可跳过**：cross-check §1 强制加载 `dianya-novel-human`，无开关，不通过就阻塞

## 七枚 Skill 详解

### 1. dianya-novel-engine（主引擎 · 总调度）

**触发短语**：写小说、创作故事、写短篇/中篇/长篇、灵感展开、小说大纲

**职责**：
- **Phase 1**：灵感展开（定人→定冲突→定结局→定切口）→ 篇幅判断 → 大纲生成
- **Phase 2**：典雅白话风格规则（古今之辨、句子/段落/篇章/对话四层、场景转换）
- **Phase 3**：交叉检查（由子引擎调用 cross-check，主引擎不重复定义）
- **Phase 4**：风格报告

**核心输出**：
- 一句话核心（全文北极星）
- 结构化大纲（按篇幅格式）
- 角色设计（命名/身份约束/关系类型/功能检查）
- 叙述者声音选择（限知第三人称/第一人称/说书人腔/评论式第三人称）
- 情绪弧线设计（五段曲线/篇幅调整/长篇情绪接力）
- 物象系统设计（主题场/空间场/季节场）

### 2. dianya-engine-short（短篇引擎）

**适用**：8,000-30,000 字，2-4 角色，无副线，一个反转撑一篇

**核心结构**：开头钩子 → 铺垫（3+线索）→ 升级（情绪加压）→ 反转（一节内揭示）→ 余韵结尾

**特色**：
- 四种钩子类型（冲突前置/反常行为/信息炸弹/代入式提问）
- 反转强度判定（信息冲击/情绪跃升/节奏突变/不可逆）
- 短篇常见失败模式诊断
- 写后强制四步（字数→朗读→交叉校验→风格报告）

### 3. dianya-engine-medium（中篇引擎）

**适用**：30,000-100,000 字，4-8 角色，1 条副线，完整起承转合

**核心结构**：起（第1-3章）→ 承（第4-8章）→ 转（第9-12章）→ 合（第13章-结尾）

**特色**：
- 连贯性保证机制（追踪文档 + 写前强制5步 + 写后强制更新 + 自检7项）
- 副线管理（只能1条，不超过4章，在转折点交汇）
- 章节标题四字强制规则
- 阶段审计（每10章全检）
- 变更管理（四步锁）
- 连贯性崩溃急救（轻/中/重三级）

### 4. dianya-engine-long（长篇引擎）

**适用**：100,000+ 字，8+ 角色，2-3 条副线，多卷本

**核心结构**：第1卷（起·格局建立）→ 第2卷（承·冲突扩大）→ 第3卷（转·危机）→ 第4+卷（合·收束）

**特色**：
- 长篇治理文件系统（宪法/规划/设定/追踪 四层目录）
- 连贯性保证机制（比中篇更严格——增加世界观检查和卷级审计）
- 字数追踪与卷预算（膨胀预警 + 超标处理）
- POV 管理与视角切换规则
- 追踪文档自我校验（伏笔密度/角色活性/章节日志质量）
- 阶段审计（每5章/10章/20章/每卷结束 分层审计）
- 长篇崩坏预防（力量失控/节奏拖沓/伏笔断裂/角色扁平/烂尾危机）

### 5. dianya-wuxia（武侠引擎）

**适用**：武侠、江湖、古龙风格

**核心替换**：
- 叙述者声音 → **评论式第三人称**（反问叙事：岂非/何尝/怎的）
- 对话规则 → 武侠对话模式（斗嘴/挑衅/哲理辩论/密谈）+ 江湖切口
- 打斗描写 → 快招速写（80%）+ 绝技慢写（20%）+ 招式命名四法
- 角色系统 → 武侠原型（游侠/剑客/掌门/隐士/魔头）+ 绝技/招牌
- 古典词密度 → 调整至 3-8%（武侠古风偏轻，重节奏轻堆砌）
- 物象系统 → 武侠中减弱，动作和对话承担情绪

**古龙风格模式**（可选整体切换或逐项混搭）：
1. 警句断言（"人生本就……"）
2. 瞬间美学（写结果不写过程）
3. 重复定型（角色特征如音乐动机反复出现）
4. 绝对化句式（"绝无/任何/永远/从未"）
5. 纯对话叙事（50-70% 对话占比）
6. 侦探推理结构（谜面→线索→揭示）
7. 存在孤独底色（哲学调性）

### 6. dianya-novel-human（去AI味引擎 · 单一真相源）

**触发短语**：去AI味、AI写作痕迹、人性化改写、文字太机器、不像人写的、AI腔太重

**核心内容**：

**35 项 AI 写作指纹**（4 级严重度）：

| 级别 | 数量 | 包含 | 判定 |
|------|------|------|------|
| 💀 致命 | 5 项 | "不是A而是B"句式、结尾升华、"仿佛/犹如"密集、"带着/透着/含着"万能搭配、"作为"开头 | 出现即判 AI —— 必须立即修正 |
| 🔴 高危 | 9 项 | "深吸一口气"、"眼中闪过一丝"、"心中涌起"、过度解释动机、均匀节奏等 | 高概率 AI 信号，累积致命 |
| 🟡 中危 | 10 项 | 排比过度、修饰过度、"缓缓/轻轻/微微"滥用等 | 影响人味但可配合上下文判断 |
| 🟢 低危 | 6 项 | 破折号过度、段落长度均匀等 | 轻微影响，建议优化 |
| 🀄 中文特有 | 5 项 | "岂不妙哉"、"……呢"AI式收尾、三字连排、"的"字滥用、"就"字泛化 | 中式 AI 写作检测专用 |

**去AI三遍法**：

| 遍次 | 名称 | 目标 | 产出 |
|------|------|------|------|
| 第一遍 | 💀致命扫描 | 清除所有致命+高危指纹 | 每处标注指纹类型→给出修正建议 |
| 第二遍 | 🫁语感润色 | 变化节奏·删除过度解释·增加具体细节 | 逐段标注修改点+修改后正文 |
| 第三遍 | 👂朗读终检 | 耳朵验证流畅度 | 标记"卡住"处→修正 |

**典雅白话七铁则**：
1. 情绪不命名——用行为展示
2. 结尾不升华——用动作/画面收束
3. 叙述者不解释角色动机
4. 古典分寸——古词 3-15% 刚好
5. 出今禁令——古典社会没有手机信号
6. 对话标签古典处理——说/道/问占 80%+
7. 一段只说一件事

**五维人味评分（50 分制）**：

| 维度 | 满分 | 评估内容 |
|------|------|---------|
| 直接性 | 10 | 直接陈述事实还是绕圈宣告？ |
| 节奏 | 10 | 句子长短是否自然变化？ |
| 信任度 | 10 | 是否信任读者的理解力？ |
| 真实性 | 10 | 读起来像真人说话吗？ |
| 精炼度 | 10 | 还有可删减的废话吗？ |

**评分标准**：45-50 优秀 → 35-44 良好 → <35 不通过（阻塞）

### 7. dianya-cross-check（交叉校验 · 调度器）

**职责**：被三个子引擎在写后阶段调用。按固定顺序调度检查模块，**不重复定义规则**——规则由各自的专项 skill 提供。

**检查顺序（强制）**：

```
§0 章节字数检查（第一道硬性门禁）
  ├─ 标题四字检查
  ├─ 单章字数判定（<3,000 → ❌阻塞，补写）
  ├─ 全篇均值检查（每5章触发）
  └─ 溢出章管理

§1 AI味深度扫描（第二道硬性门禁 · 强制加载 dianya-novel-human）
  ├─ 35项AI指纹全景扫描
  ├─ 典雅白话七铁则验证
  └─ 五维人味评分（<35 → ❌阻塞）

§2 古风味觉（6项评分）
§3 作家风格对标（8维测量）
§4 连贯性检查（角色/伏笔/时间线/世界观/文风/节奏）
§5 短篇专项（禁用词密度/钩子力度/反转有效性）
§6 对话专项（身份一致性/留白质量/标签使用/节奏）
§7 武侠专项（反问叙事/打斗节奏/招式命名/力量平衡/古龙风）
```

**判定机制**：任何 ❌ → 阻塞（修正后才能继续）。⚠️ → 建议修正，不阻塞当前输出。

## 安装步骤

### 前置条件

- **Claude Code**（CLI 或 IDE 插件）已安装并登录
- 操作系统：Windows / macOS / Linux 均可

### 步骤 1：克隆或复制 Skill 文件

将 7 个 skill 目录复制到 Claude Code 的 skills 目录：

```bash
# Claude Code 默认 skills 目录
# Windows: C:\Users\<用户名>\.claude\skills\
# macOS/Linux: ~/.claude/skills/

# 如果你已通过 Claude Code 的 skill 系统安装了这些 skill，
# 它们会自动出现在上述目录中。

# 手动安装方式：
cp -r dianya-novel-engine/ ~/.claude/skills/
cp -r dianya-engine-short/ ~/.claude/skills/
cp -r dianya-engine-medium/ ~/.claude/skills/
cp -r dianya-engine-long/ ~/.claude/skills/
cp -r dianya-wuxia/ ~/.claude/skills/
cp -r dianya-novel-human/ ~/.claude/skills/
cp -r dianya-cross-check/ ~/.claude/skills/
```

### 步骤 2：验证安装

在 Claude Code 中输入以下命令确认 skill 已被识别：

```
/list-skills
```

你应该能在输出列表中看到全部 7 个以 `dianya-` 开头的 skill。

### 步骤 3：验证调用

输入以下测试命令，确认主引擎可以被正确触发：

```
典雅白话 短篇：一个从不说话的郎中，在某天深夜被人敲开了门
```

如果 Claude 开始执行灵感展开（定人→定冲突→定结局→定切口）并产出大纲，说明引擎加载成功。

## 使用方式

### 基础用法：按篇幅启动

启动写作的最简方式是指定篇幅 + 灵感：

```
典雅白话 短篇：<一句话灵感/梗概>
典雅白话 中篇：<一句话灵感/梗概>
典雅白话 长篇：<一句话灵感/梗概>
```

示例：

```
典雅白话 短篇：旧宅里的老钟每到子时就停，停了整整十年，
有一天它忽然又走了——走的是倒针
```

### 武侠写作

武侠题材会自动加载 `dianya-wuxia`：

```
典雅白话 武侠短篇：<灵感>
典雅白话 武侠中篇：<灵感>
典雅白话 武侠长篇：<灵感>
```

古龙风格模式：

```
典雅白话 武侠中篇 古龙风：<灵感>
```

部分技法混搭：

```
典雅白话 武侠中篇：<灵感> +古龙打斗 +古龙句式
```

### 写特定章节

```
典雅白话 中篇写第3章
典雅白话 长篇写第12章
```

### 仅校验（不写新内容）

```
典雅白话 中篇校验第5章
典雅白话 长篇校验第3章
典雅白话 武侠检查：<文本>
```

### 阶段审计（长篇专用）

```
典雅白话 长篇轻检          # 每5章
典雅白话 长篇审计          # 每10章
典雅白话 长篇卷审计        # 每卷结束
典雅白话 长篇字数盘点
典雅白话 长篇变更：<变更内容>
```

### 去AI味独立使用

## 写作范例

以下是一段按照引擎生成的典雅白话文字（短篇开篇）：

```
沈栀嫁进顾家的第三日，才知道夫君有个住在别院的"表妹"。

是管家说漏了嘴。他说漏嘴的时候正在给她倒茶，水柱一偏，
浇在案上，他慌忙拿袖子去擦。沈栀没有追问。她只是把桌上的
水渍看着，看它慢慢浸进木纹里。

第三天夜里，她问顾淮之。

"明日我去别院看看。"她的语气像在说天气。

顾淮之手里的茶盖落在碗沿上，叮的一声。他没说话。隔了很久，
久到灯花爆了两次，他才说了一句："路不好走。"

"路不好走，"沈栀重复了一遍，"我走着看看。"
```

**这段文字通过了什么检查？**
- ✅ 无 AI 腔句式（无"不是A而是B"/"仿佛"/"作为"）
- ✅ 情绪不命名（沈栀的怀疑和决心通过动作展示）
- ✅ 结尾不升华（"我走着看看"是动作，不感慨）
- ✅ 古词密度 ~6%（茶盏/案/灯花/碗沿——自然融入，不堆砌）
- ✅ 对话标签只用"说""道"类（"问""说""重复了一遍"）
- ✅ 一段只说一件事
- ✅ 五维人味评分 ~46/50

## 常见问题

### Q：这 7 个 skill 是必须全部安装的吗？

**A**：是的。主引擎（`dianya-novel-engine`）是入口，三个子引擎（short/medium/long）是执行端，cross-check 是质量门禁，human 是去AI单一真相源，wuxia 是武侠专项。缺一个都会导致调用链断裂。

但不写武侠题材的话，`dianya-wuxia` 不会被触发——装了就行，不会影响非武侠写作。

### Q：短篇/中篇/长篇的判断是自动的吗？

**A**：默认自动判断——主引擎 Phase 1 会根据核心冲突复杂度、角色关系网、时间跨度、是否需要世界观铺陈、是否需要副线这五个维度综合判断。但你也可以手动指定篇幅（`典雅白话 短篇：...`），指定后跳过自动判断。

### Q：我可以只用去AI引擎（dianya-novel-human）检查已经写好的文章吗？

**A**：可以。`dianya-novel-human` 是独立可用的——直接粘贴文本，它会执行完整的 35 项指纹扫描 + 七铁则验证 + 五维评分。不需要先经过其他引擎。

### Q：交叉校验的 ❌ 阻塞如果我不认可怎么办？

**A**：阻塞判定是硬性的——AI 致命指纹和字数不足是客观指标。但你可以告诉 Claude 你的具体理由（比如某句话在你的上下文中有特殊用意），Claude 会根据实际情况调整判定。规则是工具，最终决定权在你。

### Q：引擎生成的文字会不会千篇一律？

**A**：不会。引擎是约束系统，不是模板。它告诉 Claude "什么不能写"（禁用句式/禁用词/情绪不命名），但"写什么"完全取决于你的灵感和 Claude 的创作。七位坐标作家 + 古龙风格模式 + 物象场选择保证了每次生成的文字都有不同的"声口"。

### Q：长篇治理文件看起来很复杂，真的需要建这么多文档吗？

**A**：如果你的长篇只有 10-20 章，可以用中篇的轻量追踪。但超过 30 章的叙事，追踪文档是救命用的——伏笔断裂、角色扁平、时间矛盾是长篇最常见的崩坏模式。治理文件的前期投入越大，后期修补的代价越小。

### Q：支持现代背景的小说吗？

**A**：部分支持。引擎定位是"古典背景的典雅白话"。如果你写现代背景但追求典雅笔法（如白先勇《台北人》），可以选择"清-民国"时代基调，现代事物可以出现但用词保持典雅。纯现代口语化的小说不在引擎的优化范围内。

## 进阶配置

### 自定义作家坐标

在 `dianya-novel-engine` 的 Phase 4 风格报告中，你可以要求引擎输出与你指定作家的对标数据。如果你偏好的作家不在七位坐标之内，可以告诉 Claude：

```
请以老舍的风格作为标杆，对标时参考他的句长、口语密度、对话节奏
```

Claude 会在本次写作会话中临时使用这个自定义坐标。

### 调校门禁敏感度

默认的门禁是标准敏感度。如果你觉得引擎太严格，可以调整：

```
本章交叉校验时，将AI味扫描放宽到仅检查💀致命级和高危级，
中低危仅记录不阻塞
```

反之：

```
本章使用严格模式——中危级也阻塞
```

### 批量回检

如果已经写了多章，想统一回检：

```
典雅白话 中篇审计    # 自动检查最近10章
典雅白话 全篇回检    # 检查所有已写章节
```
