# 典雅白话小说引擎

**将任意灵感转化为典雅白话风格小说的完整引擎。**

> 核心信念：最好的中文小说，读起来既不像翻译腔，也不像AI腔——它像一个人在对你娓娓道来，每个字都恰到好处。

---

## 是什么让一段文字像"典雅白话"？

典雅白话不是文言文，不是大白话，不是翻译体，不是公文腔。它是**四个否定之后剩下的那个东西**：

| 排除项 | 特征 | 反面例子 |
|--------|------|----------|
| 文言腔 | 滥用"之乎者也"、刻意古雅 | "然则今日之事，岂非天命乎？" |
| 翻译体 | 欧化长句、被动语态、"作为"开头 | "作为一位资深教师，他被他学生的行为深深地感动了。" |
| 公文腔 | 空洞抽象、套话连篇 | "进一步提升综合素养，切实增强责任意识" |
| AI腔 | 过度工整、万能搭配、解释充分 | "不是愤怒，而是深深的失望。他深吸一口气，眼中闪过一丝复杂的情绪。" |

**典雅白话 = 精准 + 自然 + 有节奏。** 每个词都有理由存在，每个句子都有呼吸，每段话都有起伏。

### 古风韵味：一条更窄的路

本引擎定位为**带古风韵味的典雅白话**——不是简单"避免现代感"，而是有意识地从古典汉语汲取词汇、意象和节奏，同时保持白话文的核心句法。

它落在这样一个光谱上：

```
文言文 ─── 古风典雅白话 ─── 普通白话 ─── 口语化 ─── 网络用语
          ↑ 本引擎的目标
```

关键区分：

| 容易混淆的概念 | 是什么 | 不是 |
|--------------|--------|------|
| **古风典雅白话** | 用古典词汇和意象写白话句子，允许少量的半文半白 | 用文言语法写文章 |
| **白话文运动初期的语言** | 鲁迅、周作人、胡适——文白夹杂有实验感 | 本引擎追求浑然一体，不露痕迹 |
| **现代古风小说** | "她唇角微勾，眸色渐深"——网络古风套话 | 扎实的古典词汇，拒绝公式化描写 |
| **半文半白** | "余忆童稚时，能张目对日"——文言 | 白话句法 + 古典用词 |

**古风韵味的操作定义**：
1. 词汇层：使用古典汉语词汇（尚未、犹、甚、即、乃、遂），但**每句话不超1-2处**
2. 意象层：取古典诗文的物象系统（竹帘、青瓷、烛火、石阶、纸伞、残荷）
3. 节奏层：继承古典散文的骈散结合——偶尔对仗，不刻意
4. 禁忌层：无现代事物、无西方句式、无网络语言、无翻译体抽象词

### 七位坐标作家

这七位（含一种风格类型）在"典雅白话"光谱上的位置不同，古风浓度亦不同。按古风韵味从浓到淡排列：

| 作家 | 古风浓度 | 可学什么 | 一句话特征 |
|------|---------|---------|-----------|
| 沈从文 | ★★★★★ | 古典词汇+白话句法，浑然一体 | 写景融情，古典意象自然流淌 |
| 白先勇 | ★★★★☆ | 现代叙事框架里的古典雅致 | 最深情的话用最平淡的语气说出来 |
| 张爱玲 | ★★★★☆ | 古典比喻资源 × 现代心理洞察 | 比喻出人意料，用词精准如刀 |
| 废名 | ★★★☆☆ | 极简中的古意 | 句子极短，留白极大，有禅意 |
| 汪曾祺 | ★★★☆☆ | 平淡中有古典韵味 | 口语入文但不俗，家常话里有功夫 |
| 阿城 | ★★☆☆☆ | 减法写作的极致 | 能用一个字不用两个字，能不说就不说 |
| **白话江湖风** | ★★☆ | 武侠白话·反问叙事·对话驱动·金句收束 | 用最短的句子写最快的剑，用反问代替抒情 |

**白话江湖风**不是一位作家，而是一种**实战验证的武侠白话写法**。它的特征：
- 段落极短（一句一段常见），节奏突刺，不留过渡句
- 对话占比 40-55%，情节主要由对话推进，人物用台词交锋代替内心独白
- 叙述者以**反问句**（岂非/何尝/怎的）介入评论——比"限知第三人称"多一层温度，比"说书人腔"少一层距离
- 章尾用**金句收束**——一个意味深长的短句定格画面，不解释不升华
- 古典词密度低（~3-5%），古典感靠**句式节奏**（对仗/四字格/反问）而非堆砌古词
- 物象使用极少，情绪由**动作和对话**共同承担
- 适用于**武侠、江湖、悬疑**等强情节类型

**重要**：沈从文的《边城》是古风典雅白话的**最高范本**——通篇白话句法，但词汇和意象全是古典的，读不出半点现代感也读不出半点文言腔。以此为基准。

## 引擎总览

本引擎按篇幅分三条子引擎。每条的结构设计、角色数量、节奏模式和交叉检查各有侧重。**武侠题材额外加载 `dianya-wuxia`，可选古龙风格模式。**

```
用户灵感（题材/人设/梗概/情绪）
  │
  ├─ 题材判断 → 武侠/江湖？
  │   ├─ 加载 dianya-wuxia（替换结构、叙述声音、角色、打斗规则）
  │   └─ 风格模式？ → 古龙风 → 启用古龙风格模式（警句断言/瞬间美学/重复定型/绝对化句式/纯对话/侦探结构/孤独底色）
  │
  ├─ 篇幅 < 3万字  → 短篇引擎
  ├─ 3万-10万字   → 中篇引擎
  └─ > 10万字     → 长篇引擎
        │
        ├─ Phase 2：典雅白话生成（三引擎共享风格规则）
        ├─ Phase 3：交叉检查（按篇幅调整检查重点，含古龙专项）
        └─ Phase 4：风格报告
```

## Phase 1：灵感展开与大纲生成

引擎的真正入口。用户给的不一定是完整梗概——可能只是一句话、一个人设、一段情绪。Phase 1 负责将其展开为可写的结构化大纲。

### 1.1 接收与补全

用户输入可能是以下任意形式：

| 输入类型 | 示例 | 需要补全 |
|---------|------|---------|
| 完整梗概 | "一个嫁入豪门的女子发现夫君有秘密" | 几乎不需补全 |
| 一句话灵感 | "旧宅里的老钟每到子时就停" | 补人物/冲突/结局 |
| 人设先行 | "一个从不说话的郎中" | 补故事/冲突/结局 |
| 情绪先行 | "想写一种很安静的悲伤" | 补人物/事件/冲突 |
| 题材/类型 | "民国背景的悬疑故事" | 全部需要展开 |

**补全顺序**（做完前一步再做下一步）：

1. **定人**：这个故事里必须有至少两个人，各想要不同的东西 → 命名，写明各要什么
2. **定冲突**：两个人"想要的东西"发生碰撞的那一点，就是核心冲突
3. **定结局**：冲突最终怎么收场？（不需要详细情节，只需要方向：谁赢了/谁变了/什么都回不去了）
4. **定切口**：故事从哪个时间点进入？从冲突爆发前/爆发时/爆发后？

补全完成后，产出**一句话核心**。这句话是整篇小说不可偏离的北极星。

> 示例："这是一个关于沈栀在嫁入顾家后发现夫君藏了一个'表妹'，她选择用自己的方式揭开真相而非等待解释的故事。"

这句话写下来之后，写作过程中每一次卡顿，都回到这句话。

### 1.2 篇幅判断

根据补全后的信息量判断篇幅：

| 判断维度 | → 短篇 | → 中篇 | → 长篇 |
|---------|--------|--------|--------|
| 核心冲突复杂度 | 一个反转能讲清 | 需要起承转合 | 多卷才能展开 |
| 角色关系网 | 2-4人单一关系 | 4-8人多层关系 | 8+人群像网络 |
| 时间跨度 | 数日到数月 | 数月至一年 | 数年到数十年 |
| 是否需要世界观铺陈 | 不需要 | 少量 | 需要完整世界建构 |
| 是否需要副线 | 不需要 | 1条 | 2-3条 |

**简便判断**：用一句话讲完故事 → 短篇。需要一段话讲完 → 中篇。需要分好几段讲完 → 长篇。

用户可手动指定篇幅（"典雅白话 短篇：..."），指定则跳过自动判断。

### 1.3 按篇幅生成大纲

**短篇大纲**（产出内容）：

```
一句话核心：_________
核心反转：表面是____，实际是____
情绪目标：读者读完应该感到____
五段规划：
  开头（500字内）：用什么钩子？□冲突前置 □反常行为 □信息炸弹 □代入式提问
  铺垫（30-40%）：3条线索分别是__、__、__
  升级（20-30%）：情绪从__递进到__
  反转（10-15%）：揭示__
  结尾（5-10%）：用什么动作/画面/对话收束？
```

**中篇大纲**（产出内容）：

```
一句话核心：_________
起（第1-3章）：切口是____，建立____
承（第4-8章）：冲突展开方向____，副线____将在第__章交汇
转（第9-12章）：转折事件____，影响____
合（第13章-结尾）：高潮____，余韵____
每章一句话规划：[第1章] ___ [第2章] ___ ...
```

**长篇大纲**（产出内容）：

```
一句话核心：_________
全书宪法：_________
终极锚点：主角最终____
卷规划：
  第1卷（起）：核心冲突____，字数预算____
  第2卷（承）：冲突扩大____，字数预算____
  ...（后续卷可在写作中调整）
每卷一句话规划。
```

大纲产完后，立即分派到对应的子引擎执行后续 Phase。

## 角色设计：古典语境中的人物

古典背景的角色与现代小说有根本不同——社会结构、身份约束、人际规则都不一样。在动笔前必须先回答三个问题。

### 命名与称谓

古典角色不止一个名字，不同关系用不同称呼：

| 名称类型 | 谁用 | 示例 | 注意 |
|---------|------|------|------|
| 姓+名 | 正式文书/叙述者 | 沈栀、顾淮之 | 叙述中最常用 |
| 字 | 同辈朋友/平级 | 待之（顾淮之的字） | 女子通常无字，世家才女偶有 |
| 小名/乳名 | 父母/长辈/夫君 | 栀儿、三丫头 | 长辈对晚辈专用 |
| 排行+姑娘/爷 | 家中仆从/外人 | 三姑娘、顾三爷 | 大家族最常用 |
| 身份称谓 | 官场/商场 | 顾大人、沈掌柜 | 按场合切换 |
| 号 | 文人自署 | 淮安居士 | 极少在叙事中使用 |

**铁则**：同一角色在不同人面前被叫不同的名字，这是古典社会的基本事实。叙述中统一用一个名字（通常是姓+名），对话中按说话人与他的关系切换称谓。

### 身份约束

古典角色的行为自由度受身份严格限制：

| 身份维度 | 约束 |
|---------|------|
| 性别 | 未出阁女子行动受限（不能独自远行/见客需屏风/书信需经手），已婚妇人略宽松。如果女主角需要高自由度→设定她为寡妇/商家女/江湖人。 |
| 阶级 | 士农工商，跨阶级互动有限制。丫鬟与小姐可以是心腹但不能是平等的"闺蜜"。阶级跃升极为困难→不能写成现代职场升迁。 |
| 家族 | 个人命运与家族绑定。婚姻是家族事务，私奔的代价是断绝关系。角色做选择时必须考虑家族后果。 |
| 年龄 | 及笄（15岁）/弱冠（20岁）是成人节点。少年角色不可做成人决策。老者的话有天然权威。 |

### 角色关系类型

古典社会的人际关系有专属模式，不能套用现代朋友/同事/恋人框架：

| 关系 | 互动模式 | 称谓 | 禁忌 |
|------|---------|------|------|
| 主仆 | 忠心但非平等，仆从的忠诚是核心戏剧资源 | 主子/大人 vs 名字/丫头名 | 不能写成现代雇佣关系 |
| 师徒 | 尊卑分明，"一日为师终身为父" | 师父/先生 vs 徒儿/学生 | 师徒恋需极度谨慎（伦理压力） |
| 同窗 | 同门师兄弟，竞争与合作并存 | 师兄/师弟/年兄 | 跨阶级同窗可能但紧张 |
| 世交 | 两家世代交好，子女从小被默认配对 | 伯父/伯母 vs 贤侄 | 利益与情感的混合 |
| 君臣 | 绝对的权力差，但忠/谏/叛是经典张力 | 陛下/大人 vs 臣/微臣 | 直谏是美德但可能致命 |

### 角色功能检查

写完角色设定后，逐一过：

1. 这个角色删掉，故事还能讲吗？不能→留。能→删或合并。
2. 这个角色想要的东西和其他角色想要的东西冲突吗？不冲突→没有戏剧。
3. 这个角色在故事结束时有变化吗？没有→考虑删，除非他是功能角色（送信/带路/提供线索）。

## 叙述者声音：谁在讲这个故事

典雅白话最微妙的选择不是词汇，而是**谁在说话**。叙述者声音决定了词汇上限、信息控制和读者的距离感。

### 三种声音

| 声音类型 | 特征 | 古风适配 | 适合篇幅 |
|---------|------|---------|---------|
| **限知第三人称**（贴角色） | 读者只看到角色看到的东西。叙述语言保持典雅，但不等于角色的内心独白。 | ★★★★★ 最推荐——节奏自由，可典雅可白话，是沈从文/白先勇/张爱玲的共同选择。 | 短中长通用 |
| **第一人称回忆录** | "我"在讲已经发生的事。天然有古意（古典笔记小说传统），但限制了信息范围——"我"不在场的不能写。 | ★★★★ 适合情感驱动的短篇和中篇。写长篇时信息受限严重。 | 短篇、中篇 |
| **说书人腔** | 叙述者是一个可见的讲述者。可用"话说""且说""这是后话"等传统套语。与读者有直接交流。 | ★★★ 需谨慎——用得好有古典韵味（《三言二拍》遗风），用多了像评书，破叙事沉浸。 | 长篇（只在章首/章尾/关键转折处用） |
| **评论式第三人称** | 介于限知和说书人之间。贴角色但不完全隐去叙述者——偶尔用**反问句**（岂非/何尝/怎的）跳出来评论一笔，随即退回角色视角。 | ★★★★ 武侠、江湖、悬疑等强情节类型的首选。有说书人的温度但不越界。 | 中长篇通用（短篇慎用——篇幅短，反问会显挤） |

### 评论式第三人称：反问叙事技法

这是白话江湖风最独特的武器。操作规则：

- **每章反问不超过3处**——多了从"品评"变成"唠叨"
- **反问只出现在情绪高点或段落收束处**——不在平铺直叙中突然反问
- **反问句的格式**：岂非……？/何尝不是……？/怎的……？/谁又……呢？
- **反问之后立即回到事件叙述**——不展开议论，不接"因为……所以……"

> 示例："人生会遇上许多的事，可是真正让人轻松而开心的事，岂非都是一些小事？"——反问停在恰好，下一段切回场景："三人一路乘着春风暖日，一路闲聊走向春风得意楼。"

### 选择与坚守

- 选定一种声音后，**全篇不换**（长篇可以在卷首/卷尾切换为说书人腔，但主体保持限知第三人称）
- 最常见的错误：限知第三人称写着写着突然跳出一句全知视角的总结（"她不知道，这将是她最后一次见到他"）→❌
- **四种声音的共同底线**：叙述者不解释角色动机，不替角色感慨

### 声音与词汇的关系

| 声音 | 可用古典词密度 | 可用"且说/却说"等 |
|------|-------------|-----------------|
| 限知第三人称 | 5-15% | 不用 |
| 第一人称回忆录 | 依角色身份定 | 不用 |
| 说书人腔 | 10-20% | 可用（有节制） |
| 评论式第三人称 | 3-8% | 不用——用反问代替传统套语 |

## 情绪弧线设计

引擎在无数地方谈情绪（不命名情绪、情绪递增、情绪强度），但不能只靠写的时候现场判断。**情绪弧线必须在写之前设计好**。

### 基本模型：五段情绪曲线

```
情绪强度
  ↑         ╱‾‾‾‾‾╲
  │        ╱   ·3·  ╲
  │    ·2·╱          ╲·4·
  │    ╱╱              ╲╲
  │  ╱╱                  ╲·5·
  │ ╱·1·                   ╲
  └──────────────────────────→ 篇幅
```

| 节点 | 位置 | 功能 | 情绪方向 |
|------|------|------|---------|
| 1. 建立 | 开篇 | 建立情感基线，让读者知道"正常"是什么样 | 平稳偏暖或偏冷（定调） |
| 2. 扰动 | 15-25% | 第一个扰动出现，情绪开始偏离基线 | ↑ 上升 |
| 3. 顶点 | 50-70% | 情绪最高点——不一定是反转，但一定是全文最强烈的情绪段落 | ↑↑ 最高 |
| 4. 回落 | 70-90% | 高潮后的余波，情绪开始沉降 | ↓ 下降 |
| 5. 余韵 | 结尾 | 最后一个情绪落点——不是归于零，而是停在某个有意味的位置 | → 平稳但不归零 |

### 按篇幅调整

| 参数 | 短篇 | 中篇 | 长篇 |
|------|------|------|------|
| 情绪节点数 | 3（建立→顶点→余韵） | 5（完整曲线） | 每卷一条曲线，卷间有情绪接力 |
| 顶点位置 | 反转段 = 顶点 | 转折点附近 | 每卷有自己的顶点，全书顶点在最后一卷 |
| 情绪变化密度 | 每500字至少一次微变化 | 每章至少一次变化 | 每2-3章一次大变化，但每章有小波动 |
| 最低情绪要求 | 不能连续2段无变化 | 不能连续2章无变化 | 不能连续3章无变化 |

### 长篇情绪接力

长篇的最大陷阱是卷与卷之间情绪断裂——第一卷结尾情绪很高，第二卷开篇又从零开始。

**卷间情绪传递规则**：
- 每卷结束时标注"卷末情绪状态"（主角A：愤怒（8/10），主角B：绝望（6/10））
- 下一卷开篇从上一卷的数值开始，不能归零
- 如果上一卷结尾情绪是"悲伤（9/10）"，下一卷开篇不能是"平静地喝茶"——要么保持悲伤，要么展示角色在压抑悲伤（那也是一种悲伤）

**定位**：一个反转撑一篇。所有内容为唯一的情绪目标服务。

### 适用范围

| 参数 | 规格 |
|------|------|
| 字数 | 8,000 - 30,000 |
| 场景数 | 5-12 个场景 |
| 角色数 | 2-4 人（主角 + 关键对手 + 1-2 配角） |
| 子线 | 无 |
| 时间跨度 | 短（数日到数月） |

### 结构：核心反转

```
开头钩子（前500字）→ 铺垫（3+线索埋设）→ 升级（情绪加压）→ 反转（一节内揭示）→ 余韵结尾
```

| 段 | 占比 | 功能 | 必备要素 |
|----|------|------|---------|
| 开头 | 5-10% | 一句话钩住读者 | 冲突前置 / 反常行为 / 信息炸弹 / 代入式提问 |
| 铺垫 | 30-40% | 建立关系，埋设线索 | 至少3条反转线索分散在不同位置 |
| 升级 | 20-30% | 冲突加剧，情绪积累 | 每节情绪强度递增，不允许连续2节无变化 |
| 反转 | 10-15% | 揭示真相 | 强度 > 前面所有节最高值，铺垫可回溯 |
| 结尾 | 5-10% | 余韵收束 | 动作/画面/短对话，不总结不感慨 |

### 写作原则

- **每句话必须有信息**：不推动剧情、不铺垫反转、不推高情绪 → 删
- **默认第一人称**：短篇的代入感最优先；特殊题材（多视角悬疑）可用第三人称
- **一个反转撑一篇**：不多线、不铺世界观，所有伏笔指向同一个反转
- **节奏紧凑**：场景切换快，不留无效过渡段落

## 中篇引擎（3万 - 10万字）

**定位**：有完整的起承转合，有副线，有角色深度。介于短篇的锐度和长篇的广度之间。

### 适用范围

| 参数 | 规格 |
|------|------|
| 字数 | 30,000 - 100,000 |
| 章节数 | 8-20 章 |
| 角色数 | 4-8 人（主角 + 2-3 核心配角 + 反派 + 1-2 功能角色） |
| 子线 | 1 条副线 |
| 时间跨度 | 中（数月至一年） |

### 结构：完整弧线

```
起（第1-3章）→ 承（第4-8章）→ 转（第9-12章）→ 合（第13章-结尾）
```

| 阶段 | 占比 | 功能 | 每章字数 |
|------|------|------|---------|
| **起** | 15-20% | 进入故事的切口 | 3,000-5,000 |
| **承** | 35-40% | 冲突展开，副线并行 | 3,000-5,000 |
| **转** | 25-30% | 转折点，危机升级 | 3,000-6,000 |
| **合** | 15-20% | 高潮 + 余韵 | 3,000-5,000 |

### 章节模板

每章遵循统一节拍：

```
章首（200-500字）：钩子/快速切入前情
  ↓
中段（70%内容）：推进核心事件，牵动副线
  ↓
章尾（200-300字）：蓄力或收束，留下悬念
```

### 副线管理

- 只能有 **1 条副线**，串联不超过 4 个章节
- 副线必须与主线在**转折点交汇**
- 副线角色不能超过核心角色的一半

### 写作原则

- **每章必须有变化**：角色状态、信息量、关系至少一个维度发生变化
- **章尾必须留悬念**：不需要大悬念，但要有"还想往下看"的牵引力
- **副线不喧宾夺主**：副线篇幅不超过全文的 20%
- **允许慢热**：中篇可以在前三章徐徐展开，不必像短篇那样第一句就炸

## 长篇引擎（10万字以上）

**定位**：多卷本、多人物的长篇叙事。需要前期治理（宪法/卷纲/阶段规划）以保持长线不崩。

### 适用范围

| 参数 | 规格 |
|------|------|
| 字数 | 100,000+ |
| 章节数 | 30+ 章 |
| 卷数 | 2+ 卷（每卷 10-30 章） |
| 角色数 | 8+ 人（主角群 + 反派势力 + 功能角色） |
| 子线 | 2-3 条副线 |
| 时间跨度 | 长（数年到数十年） |

### 结构：多卷弧线

```
第1卷（起点·格局建立）→ 第2卷（发展·冲突扩大）→ 第3卷（转折·危机）→ 第4+卷（高潮·收束）
```

| 卷 | 功能 | 字数 | 核心任务 |
|----|------|------|---------|
| 1 | 建立世界+人物 | 20-30万 | 让读者知道"这是一个什么样的故事" |
| 2 | 展开冲突 | 20-30万 | 把核心矛盾的复杂面展示出来 |
| 3-N | 转折+高潮 | 视需要 | 不断升级，不停创造新的悬念 |
| 最后一卷 | 收束 | 10-20万 | 解决核心矛盾，不要烂尾 |

### 长篇治理文件

超过30章的长篇需要维护以下文档：

```
项目/
├── 宪法/                 # 不可改动的设定（终极目标、核心规则）
│   ├── 全书宪法.md       # 一句话全书写什么
│   ├── 终极锚点.md       # 主角走到终点的状态
│   └── 世界观基石.md     # 世界运转的基本法则
├── 规划/                 # 弹性规划，可随正文调整
│   ├── 卷纲.md           # 每卷的核心冲突和起止
│   ├── 阶段规划.md       # 每10-20章的节奏安排
│   └── 变更日志.md       # 每次结构性改动的记录
├── 设定/                 # 场景/人物/物品，按需翻阅
│   ├── 角色/             # 每人一个文件
│   ├── 地理/             # 关键地点
│   └── 物品/             # 重要道具
├── 正文/
│   ├── 第1章_章名.md
│   └── ...
└── 追踪/
    ├── 伏笔.md           # 已埋设/已回收的伏笔
    ├── 时间线.md         # 故事内时间轴
    └── 角色状态.md       # 每个主角的当前状态
```

### 章节写作原则

- **一章一个核心事件**：不超过2个事件，不分心
- **每10-20章做阶段审计**：检查伏笔回收、节奏、情绪曲线
- **每卷结束做卷审计**：总结本卷得失，调整下卷规划
- **章尾制造牵引力**：悬念/未解决的张力/新信息
- **长篇容许闲笔**：10章中可以有一章"慢"，但不能连续两章都慢
- **长篇容许多线并行**：但每章紧贴一个POV，不乱切视角

### 常见长篇崩坏预防

| 崩坏类型 | 症状 | 预防 |
|---------|------|------|
| 力量失控 | 主角太强，没有悬念 | 设定明确的代价或限制 |
| 节奏拖沓 | 连续5章无进展 | 阶段审计会发现，及时调整 |
| 伏笔断裂 | 前面埋的线后面忘了 | 维护伏笔追踪表 |
| 角色扁平 | 写多了发现角色没有成长 | 每卷检查角色状态是否有变化 |
| 烂尾危机 | 收不了场 | 开始写之前就有最终锚点 |

## 连贯性保证机制（中篇·长篇强制规则）

> ⚠️ **强制规则**：中篇（3万-10万字）和长篇（10万字+）在写作过程中必须执行以下连贯性机制。这是引擎的硬性门禁，不执行不得进入下一章写作。

### 一、必建追踪文档（写作前建好）

中篇和长篇在写第一章之前，必须先建以下追踪文档。篇幅达到即自动触发，不依赖用户提醒。

| 文档 | 中篇强制 | 长篇强制 | 功能 |
|------|---------|---------|------|
| **伏笔登记表** | ✅ | ✅ | 记录每章埋了哪些伏笔，计划何时回收 |
| **角色状态表** | ✅ | ✅ | 记录每个主角当前的身份/能力/关系/心理状态 |
| **时间线** | ✅ | ✅ | 故事内的时间推进，避免时序矛盾 |
| **章节日志** | ✅ | ✅ | 每章一句话摘要 + 本章结束时的状态快照 |
| **变更日志** | — | ✅ | 记录任何结构性改动（设定变更/角色新增/情节调整） |

**伏笔登记表格式**（每新增一个伏笔就追加一行）：

```
| 伏笔ID | 埋设章节 | 内容 | 计划回收章节 | 状态 | 重要度 |
|--------|---------|------|------------|------|--------|
| F001 | 第3章 | 女主手上有一枚成色古怪的玉佩 | 第12章 | 已埋 | 高 |
| F002 | 第5章 | 管家提到老宅地窖门锁换了 | 第15章前后 | 已埋 | 中 |
```

**角色状态表格式**（每章更新涉及的角色）：

```
| 角色 | 当前身份 | 当前位置 | 关键关系状态 | 心理状态 | 上次更新章节 |
|------|---------|---------|-------------|---------|------------|
| 沈栀 | 沈家三小姐 | 京城娘家 | 与顾淮之冷战 | 迷茫但倔强 | 第8章 |
| 顾淮之 | 翰林院编修 | 京郊别院 | 与沈栀冷战 | 后悔但不肯低头 | 第8章 |
```

**章节日志格式**（每写完一章立即写入）：

```
第N章：章名
一句话摘要：{发生了什么}
本章结束时角色状态：{主角A→状态X，主角B→状态Y}
新埋伏笔：{F00X}
已回收伏笔：{F00Y}
下章展望：{下一章要写什么}
```

### 二、每章写作前的强制步骤

写任何一章（中篇的第2章起，长篇的全部章节）之前，必须先执行以下操作。少一步不得动笔。

**步骤1：读上一章的章节日志**
确认上一章结束时角色在什么状态、故事停在什么位置。如果上一章日志不存在，必须先补写，不得跳过。

**步骤2：查伏笔登记表**
- 确认有哪些待回收的伏笔即将到期
- 确认本章不要提前泄露未到回收时机的伏笔
- 如果发现某条伏笔超过计划回收章节5章还未回收 → **标记为逾期伏笔**，本章必须安排回收或推进

**步骤3：查角色状态表**
- 确认本章涉及的角色当前状态
- 检查角色状态是否有被前文改变的（关系变化/身份变化/位置变化）
- 如果发现角色状态与上一章矛盾 → **必须修正，不得带着矛盾写下去**

**步骤4：查时间线**
- 确认当前故事时间
- 确认本章的时间推进合理（不跳跃到未经过的时间点，不出现"今天"和"三天后"矛盾的叙述）

**步骤5：一句话确认本章意图**
写之前用一句话说清本章要完成什么。示例：
> "第9章：顾淮之发现玉佩的来历，这会让他在第10章做出关键选择。"

如果说不清本章要干什么 → **说明这章不该现在写，先调整大纲再动笔**。

### 三、每章写作后的强制步骤

**步骤1：更新章节日志**
按上述格式写入摘要、角色状态快照、伏笔变动。

**步骤2：更新伏笔登记表**
- 新增的伏笔 → 追加新行
- 已回收的伏笔 → 状态改为"已回收"，标注回收章节
- 更改计划的伏笔 → 更新"计划回收章节"栏，在变更日志中记录

**步骤3：更新角色状态表**
凡本章涉及的角色，更新其状态。状态没有变化也要注明"无变化"——不更新的后果是下一章忘记角色已经变了。

**步骤4：快速连贯性自检（写后必答）**

| 检查项 | 中篇 | 长篇 |
|--------|------|------|
| 本章角色行为与前一章的状态是否一致？ | ✅ | ✅ |
| 本章提到的事件/物品/人物在前文已有交代？ | ✅ | ✅ |
| 时间线是否连续无跳跃或矛盾？ | ✅ | ✅ |
| 对话中提到的过去事件是否与已写章节一致？ | ✅ | ✅ |
| 本章新埋的伏笔是否登记了？ | ✅ | ✅ |
| 本章回收的伏笔是否确实是在前文埋的？ | — | ✅（长篇伏笔线长，容易记错） |
| 世界观法则是否被本章的事件违反？ | — | ✅（长篇设定多，容易崩） |

**任何一项不过 → 修正后再进入下一章。不允许累积问题。**

### 四、阶段审计规则

| 触发条件 | 中篇 | 长篇 |
|---------|------|------|
| 每写完5章 | — | 轻量检查：伏笔登记表 + 角色状态表是否在更新 |
| 每写完10章 | + 连贯性全检 | + 连贯性全检 + 节奏评估 |
| 每写完20章 | —（中篇通常已完） | + 卷审计：角色弧线 + 伏笔密度 + 副线交汇情况 |
| 每卷结束 | —（中篇不分卷） | **强制卷审计**：总结本卷，调整下卷规划 |
| 任何结构性改动前 | 必须先更新变更日志 | 必须先更新变更日志 + 做连贯性影响评估 |

**阶段审计必须产出书面结果**，写在章节日志末尾或单独的审计记录中，不得只在脑子里过一遍。

### 五、变更管理

小说写到一半需要改设定/加角色/调情节时：

1. **先写变更日志**：说明为什么改、改了什么、影响哪些章节
2. **评估回波及范围**：改动影响已经写完的章节吗？需要回溯修改吗？
3. **更新所有相关追踪文档**：伏笔登记表（新增或废弃的伏笔）、角色状态表（改设定的角色）、时间线（如果时序变化）
4. **只有做完以上三步，才能写改了之后的章节**

> **核心原则**：写作过程中唯一不允许的是"悄悄改、不记录"。改了没记录 = 后面一定会写出矛盾。

### 六、连贯性崩溃的急救措施

如果已经出现连贯性问题（角色状态矛盾/伏笔丢失/时间线冲突）：

| 严重程度 | 症状 | 处理方式 |
|---------|------|---------|
| 轻度 | 角色名字写错/地点名称不一致 | 直接回溯修正，不需停写 |
| 中度 | 伏笔超过5章未回收/角色状态自相矛盾 | 停写，回读相关章节，修正追踪文档后再继续 |
| 重度 | 情节走向与已写章节矛盾/世界观规则被打破 | 停写，做阶段审计，重写矛盾章节或调整后续大纲 |

**重度问题未解决前，禁止继续往下写新内容。**

### 古今之辨：用古典词汇，避现代事物

这是本引擎最核心的语言层规则。分两条线：**入古**（用什么古典词汇）和**出今**（避开什么现代事物）。

**入古指南（可用·当用）**：

| 古典词 | 何时用 | 何时不用 |
|-------|--------|---------|
| 尚未/未曾/未 | 描述未发生之事 | 日常对话中说"还没"更自然 |
| 犹/尚 | 表示"仍然" | 角色内心用"还"也不违和 |
| 甚/颇/极/至 | 表程度，"甚好""极静" | 连续用会显文，一句只放一处 |
| 即/乃/遂/因/故 | 表承接或因果 | 句首可用，句中有更自然的连词时不用 |
| 第……/……者 | "居首位者""次第" | 超过两处就变文言腔 |
| 须臾/俄顷/少时/良久 | 表时间，精准且雅 | 选一个合适的，不要密集出现 |

| ——（破折号） | 见下方「破折号使用标准」——仅限三类场景可用 | 克制使用，能不用则不用，目标密度 2~7/k |
| ……（省略号） | 古典白话常用标点制造节奏 | 可用，不必回避 |

### 破折号（——）使用标准

**核心原则：克制使用，能不用则不用。** 目标密度：每千中文字符 2~7 个破折号。

#### 可用场景（可用，非必用）

以下三类场景中破折号可以保留，但需逐句判断——仅当去掉后会损害语气、节奏或含义时才用。

**1. 截断语**
对话或叙述中被截断、留白、换气的瞬间。
> "师父——"  
> "你来找我——会死吗。"

**2. 警句 / 隐喻延展**
前句铺垫意象，后句用破折号延展出更深层的含义，形成「画面→真相」的节奏。
> 话是钉子——说出去就钉进墙里。  
> 一把刀用得越久——越像握着它的那个人。  
> 树死了——树根还没死。

**3. 主题揭示**
章末或段落结尾处，破折号引出全段/全章的核心落点。
> 他就明白了——他从来没有离开过江湖。  
> 刀也可以用来告诉一个人——你没有被放弃。

#### 不应使用的场景

- **"不是X——是Y"判断句式**：用逗号。~~"不是热的——是冷的。"~~ → "不是热的，是冷的。"
- **普通因果/顺承**：用逗号或句号。
- **普通补充说明**：用逗号或冒号。
- **机械列举**：用逗号分隔。
- **普通对话引入**：用冒号或逗号。
- **单纯动作描写衔接**：用逗号。

#### 编辑流程

1. Grep 全文破折号，逐句读上下文
2. 判断属于哪种类型：可用三类 → 考虑保留；其他 → 去除
3. 即使属于可用三类，也问一句："去掉后意思变了吗？"——没变就去掉
4. 编辑后复查密度，确保在 2~7/k 范围内

**古今分寸口诀**：宁用"尚未"不用"还没"，但绝不用"尚未……也"的文言对仗；宁用"甚静"不用"非常安静"，但不写"其静甚矣"。

**古典词汇使用频率分层**（防均匀撒胡椒面）：

| 层级 | 使用频率 | 典型词 | 用法说明 |
|------|---------|--------|---------|
| **高频安全** | 每章可多次出现 | 尚未/未曾、甚/极、遂/便、唯/但、良久/少时 | 融入白话最自然，读者不觉"文"。放心用，不连续两句出现即可。 |
| **中频点缀** | 每章1-2次 | 犹/尚（仍然义）、颇、即（便义）、须臾/俄顷、故（所以义） | 有意识地在情绪高点或画面转折处用。一处就够了——少才有效果。 |
| **低频慎用** | 全书只用几次 | 之（定语标记）、矣、哉、者…也句式、"其"做主语 | 用前自问：删掉字句意变了吗？没变→删。变了也未必值得留。 |

**分层自检**：写完一章后扫一遍，中频词出现在3个以上段落→太密，把多余的换成白话对应词。

### 第2层：段落层——典雅白话语感 + 古风韵律

四条核心原则，加上古风韵味的特殊技法。

**原则1：一段只说一件事**

一段只承载一个动作/一个信息/一个画面变化。一个自然段超过60字，拆。

```
反面：他推开门走进房间，看到窗边坐着一个穿红裙子的女人，她正在看书，
阳光从她背后照过来，在她头发上镀了一层金色，他忽然觉得这一幕似曾相识，
心里涌起一阵说不清的悸动。

正面：他推开门。
窗边坐着个女人，红裙子。
她在看书，光从背后照过来。
他愣在门口。
```

**原则2：情绪不命名，用行为展示**

反面："他很愤怒"
正面："他一把掀翻了桌子"

反面："她很失望"
正面："她把信叠好，放回抽屉，关了灯"

反面："他感到温暖"
正面："他没说话，把杯子握在手心"

**原则3：不要解释角色动机**

角色做了什么，让读者自己判断为什么。角色不会（也不该）把自己的动机说清楚。

反面："她决定离开他，因为她受够了他的冷漠，但她又有些犹豫，毕竟他们在一起五年了。"
正面："她把钥匙放在玄关的鞋柜上。"

**原则4：结尾不升华**

段尾不要总结、感慨、点题。用动作/对话/场景收尾。

反面："他知道，从这一刻起，一切都不同了。人生就是这样，总是在不经意间转向。"
正面："他把烟掐灭，站起来，走了出去。"

**原则5（古风专项）：善用骈散节奏**

古典散文最珍贵的遗产不是对仗，而是**骈散结合**——偶尔的对仗给文字以韵律感，大多数句子保持散句的松弛。

```
骈（对仗）："庭前柏树子，池中莲花影"——两句结构相同
散（自由）："和尚指了指标，没有说话"——完全自由的句法
骈散结合："他坐在窗前，听了一夜的雨。雨声时密时疏，像人的心事。天亮时，檐水还滴着，滴滴答答。"——起句散，中间一个半骈半散的句子收束，末句散
```

操作方法：
- 一篇文章/一章中，**刻意对仗不超过2处**，且不连续
- 更高级的用法：**意对而句不对**——意思上呼应，句子结构不同（"他在东厢读书，她在西窗绣花。"——意思对仗，句子长短不同）
- 每段在节奏上做到：长句(20-40字)+短句(3-10字)交替，松弛有度

**原则6（古风专项）：取古典意象系统**

用物象说话，不跳出来抒情。古典诗词最擅长"以物写情"——选对物象，情绪自然传达。

```
用"残荷"替代"伤感"
用"烛火"替代"温暖或希望"
用"青石板"替代"岁月痕迹"
用"纸伞"替代"疏离或守护"
用"竹帘"替代"隔阂或私密"
用"石阶"替代"等待或修行"
用"薄雾"替代"迷惘"
用"炉灰"替代"熄灭或余温"
用"明月"替代"思念或永恒"
```

使用方法：
1. 一个物象出现，读者自己会感受到情绪——**不需要额外说明**
2. 物象提前出现，后面呼应，形成结构感
3. **避免堆砌**：一段中不超2个物象，全文物象有系统性（比如都以庭院为主题，不东拉西扯）
4. 注意时代感：选择古典故事中本来就存在的物象，不要出现"他望着屏幕，像望着月光"这种错位

**古风段落写作示例**：

```
沈从文式（典范）：
"黄昏时，对河的店铺点了灯。河水在灯影里流着，流得很慢。她站在码头上，看船一只一只地靠岸，又一只一只地离去。天全黑了，她还没有走。"

——纯白话句法，但"黄昏""灯影""河水""码头""船"全是古典意象，没有一个现代词，读起来自然流畅。
```

### 物象系统设计（古风专项）

原则6教了用物象替代抒情，但如果每段随手抓一个物象，全篇读起来就像"意象杂货铺"——残荷下面放烛火、烛火旁边搁明月，互不相干。好的古风小说有一个**统一的物象场**。

**为单篇小说设计物象场，三选一**：

| 物象场类型 | 说明 | 示例 | 适用 |
|-----------|------|------|------|
| **主题场** | 所有关键物象都从一个核心主题衍生 | 主题"水"→江水/雨水/茶水/泪水/酒/墨汁 | 中短篇首选 |
| **空间场** | 物象由故事的主场景决定 | 庭院→竹帘/石阶/残荷/青瓷/银杏/棋局 | 中长篇合适 |
| **季节场** | 物象跟随故事发生的季节流转 | 秋→落叶/薄雾/炉火/雁阵/残菊/霜 | 时间跨度短的通用 |

**物象系统设计的三个步骤**：

1. **定基调物象**（全文只用1-2个核心物象反复出现，形成结构感）——沈从文《边城》的"水"和"船"贯穿始终。
2. **分配场景物象**（每个关键场景安排1-2个派生物象）——庭院故事中，早晨的竹帘、午后的石阶、黄昏的银杏、深夜的烛火。
3. **物象呼应**（同一个物象在不同章节以不同状态出现）——开篇银杏正绿，中段叶黄，结尾落了满地。物象变化即时间流逝，不需要写"三个月过去了"。

**物象自检清单**：
- 全文物象可以归到一个主题/空间/季节下吗？不能→东拉西扯。
- 有物象在全文出现2次以上吗？没有→没有结构感。
- 有物象在开头和结尾都出现吗？没有→考虑让一个物象形成回环。

### 第3层：篇章层——按篇幅调整节奏

**三拍节奏法**（每章/每段通用）：

| 拍 | 占比 | 功能 |
|----|------|------|
| 起拍 前20% | 钩子/切入 | 一句话让读者想往下看 |
| 中拍 60% | 推进/变化 | 事件推进，角色状态变化 |
| 落拍 后20% | 蓄力/收束 | 要么埋钩子，要么有余韵 |

**按篇幅的节奏差异**：

| 节奏维度 | 短篇引擎 | 中篇引擎 | 长篇引擎 |
|---------|---------|---------|---------|
| 平均每章字数 | 整篇不分章 | 3,600-5,000字 | 3,600-5,000字 |
| 起拍速度 | 100字内必须入题 | 500字内入题 | 一章内入题即可 |
| 可接受的慢开场 | ❌ 不允许 | ✅ 前三章可慢 | ✅ 卷一可慢 |
| 钩子密度 | 每300字一次 | 每章一次+章尾 | 每章章尾+卷尾大钩 |
| 闲笔（意境/氛围） | ❌ 几乎不允许 | ✅ 每个阶段1-2处 | ✅ 每卷可安排一章 |
| 视角切换 | 单一视角 | 最多2个视角 | 多个视角，一章一个 |

### 章节字数标准（强制规则）

> ⚠️ 中篇和长篇引擎启用此规则。短篇不分章，以场景为单元，参照基准线执行。

**字数统计方法（中文出版惯例）**：

```
字数 = 汉字 + 中文标点符号
```

| 计入 | 不计入 |
|------|--------|
| 汉字（CJK统一表意文字） | 西文空格、换行符 |
| 中文标点：，。！？；：、"”''【】《》（）—… | Markdown 格式标记（# * \| ` -） |
| 全角标点（U+3000-303F, U+FF00-FFEF） | 西文标点（英文逗号、句号、引号等） |
| 中文引号、省略号、破折号 | 章节标题行、技法说明/注释行 |
| 阿拉伯数字（计入，占一个中文字位） | 纯西文单词/缩写 |

> **简便验证法**：用正则 `[一-鿿㐀-䶿　-〿＀-￯—…"“]` 匹配，匹配到的字符数即中文出版字数。Node.js 验证：`text.match(/[一-鿿㐀-䶿　-〿＀-￯—…“”]/g).length`

| 参数 | 标准 | 说明 |
|------|------|------|
| **基准线** | 3,600 字 | 每章目标字数——无特殊剧情时尽量接近此值 |
| **下限** | 3,600 字 | 低于此值→章节内容不足，需检查是否缺少事件推进或情绪变化 |
| **上限** | 5,000 字 | 常规上限——超出需有明确理由（打斗高潮/关键转折/多线交汇） |
| **溢出允许** | 5,000+ 字 | 仅限关键剧情章节（转折点/终战/大揭秘），且仅单章有效——溢出一章后下一章必须回到基准线附近 |
| **平均约束** | ≤ 4,000 字/章 | 全篇完工时统计——总字数/总章数不超过4,000字 |

**字数超标分级**：

| 超标程度 | 判定 | 处理 |
|---------|------|------|
| ≤ 5,000 字 | ✅ 通过（常规范围内） | — |
| 5,000-5,500 字 | ⚠️ 需说明理由 | 章节日志中注明溢出原因，下一章收紧至 ≤3,600 字作为平衡 |
| 5,500-6,000 字 | ⚠️ 强警告 | 检查是否有多余场景/重复对话/冗余描写——必有可压缩之处 |
| > 6,000 字 | ❌ 不通过 | 必须拆分——或拆为两章，或压缩至 5,000 字以内 |

**字数不足判定**：

| 不足程度 | 判定 | 处理 |
|---------|------|------|
| 3,000-3,599 字 | ⚠️ 需检查 | 本章是否有事件推进？角色状态是否有变化？若两项皆有→通过，否则补写 |
| < 3,000 字 | ❌ 不通过 | 内容不足——事件未展开或情绪未到位，需补写至 ≥3,600 字 |

**每章写作后字数检查**（加入写后自检步骤）：

1. 统计本章正文字数（不含标题行和技法说明）
2. 对照上述标准判定
3. 超标/不足→在章节日志中记录原因和处理方式
4. 全篇每写满 5 章→做一次字数均值检查：均值 > 4,500 字→后续章节收紧至 ≤3,600 字；均值 < 3,300 字→后续章节需补足内容

**章尾四式**（选一）：

- **动作收束**：一个干净的动作结尾，让画面定格
- **对话收束**：一句让人回味的对话收尾
- **转折收束**：新信息出现，但不展开
- **金句收束**：叙述者给出一句有分量的短判断，意味深长但不解释不升华。如"而她自己就是那把剪刀。""她不知道这扇门跨进去，要三年后才能再走出来。"——这是评论式叙述者的专属武器

**不用的结尾方式**：
- ❌ 总结感慨型："这一刻，他终于懂了……"
- ❌ 升华型："也许，这就是成长的意义"
- ❌ 解释型："他不知道，更大的风暴还在后面……"

### 悬念四型

引擎频繁提到悬念/钩子/牵引力。不同类型的悬念适用于不同篇幅和位置：

| 悬念类型 | 读者心里的问题是 | 最佳位置 | 释放节奏 |
|---------|----------------|---------|---------|
| **谜题式** | "真相是什么？" | 短篇核心、中长篇主悬念 | 全篇积累→结尾一次性释放 |
| **命运式** | "他/她会怎样？" | 中长篇角色线 | 逐步释放，每章近一步或远一步 |
| **关系式** | "这两个人会怎样？" | 中长篇副线 | 低强度持续，关键时刻跃升 |
| **氛围式** | "这里不对劲……" | 开篇/章节开篇 | 快速建立→快速解答（一章内） |

**使用规则**：
- 一篇小说至少有一个主悬念（谜题式或命运式选一个），从头串到尾
- 氛围式悬念不能单独撑一篇——它是佐料，不是主菜
- 关系式和命运式可以并存（中长篇），但必须有一个更主导
- 所有悬念在结尾时必须给出交代（未必是解答，但必须有回应）

## 第4层：对话专项——什么人说什么话

对话占小说的25-45%，是典雅白话最容易泄底的地方——一句现代化对话，前面铺的所有古风氛围全破。

### 对话三原则

**原则1：身份决定用词**

| 角色身份 | 用词特征 | 示例 |
|---------|---------|------|
| 读书人/官宦 | 偶用书面词，句子偏长，有理有据 | "此事容我再思量一二。" |
| 商贩/市井 | 短句，白话为主，偶有行话切口 | "这位爷，您瞧着给。" |
| 仆从/下人 | 称谓规矩严，多用"您/老爷/姑娘" | "姑娘，外头落了雨。" |
| 武人/江湖 | 直来直去，少修饰，偶有粗豪 | "走不走？一句话。" |
| 老者/长辈 | 语速缓，惯用"从前/早年/那时候" | "那年的事，你们都还小。" |
| 少年/幼童 | 句子短，问题多，用词简单 | "娘，外头下雨了吗？" |

**原则2：对话不说透（留白）**

人物对话不是信息传递——信息传递用叙述。对话的功能是**展示人物在特定时刻的反应**。好对话的三种留白方法：

- **所答非所问**：角色回避关键问题，用无关的话岔开。读者自己感受到回避。
- **话只说一半**：角色开了头又停下。停下的地方正是重要的地方。
- **重复中的变化**：同一个问题被问两次，角色的回答不一样——变化本身说明一切。

> 反面："你还爱我吗？" / "不爱了。"（直白无味）
> 正面："你还爱我吗？" / 她没有回答。她把茶杯转了转，"明儿要落雨。"（逃避本身即答案）

**原则3：对话标签的古典处理**

| 标签 | 使用场景 | 频率 |
|------|---------|------|
| 说 | 中性通用，最安全 | 60%+ |
| 道 | 偏书面，正式场合或长者 | 20% |
| 问 | 问句专用 | 按需 |
| 无标签 | 两人对话，身份明确时直接写对话 | 可穿插2-3轮 |
| ❌ 禁用 | "低声道/轻声道/柔声道/冷声道"——副词修饰"说"是翻译体 | 用动作替代 |

**"×道"的陷阱**：不要用"笑道/叹道/怒道/喝道/答道/问道/说道"——这是最隐蔽的翻译腔。"他笑道"→"他笑了一下。……"（动作在前，对话另起）。

### 对话节奏控制

- 紧张对话：回合短（每人1-2句），切换快，无叙述穿插
- 抒情对话：回合长（允许3-4句），中间可穿插动作/环境
- 闲笔对话：一两个回合即止，不拖

## 第5层：场景转换——古典叙事的过渡技法

古典白话小说有成熟的场景转换技巧（"且说""却说""话分两头"），现代典雅白话需要化用而非套用。

### 四种转换方式

**① 空行切换（最常用）**：一个空行 = 时间/空间/视角的切换。干净，不需要连接词。

```
她关了窗。
[空行]
第二日天还没亮，院子里有了脚步声。
```

**② 物象过渡**：用同一个物象连接两个场景，形成视觉连续性。

```
他放下茶盏，盏里的茶水还荡着。
[下一段]
茶水不荡了。他已经站在了城门外。
```
——茶盏的"荡"和"不荡"完成了时间的跳跃。

**③ 声音过渡**：一个声音在A场景响起，下一个场景由此开始。

```
屋里传来一声咳嗽。
[下一段]
第二天清晨，她听见隔壁也在咳嗽。
```

**④ 叙述者收束（中长篇适用）**：用一句简短的叙述收束前文，开启下一场景。这是"且说"精神的现代传承。

> "这件事之后，他有三个月没去渡口。"
> "沈栀不说，顾淮之也不问。就这样过了十来天。"

**化用传统而不套用**：
- ✅ 可用："这且不提。""这是后话。"
- ❌ 慎用："且说""却说""按下不表"（太像说书人，破叙事沉浸）
- 原则：传统套话只在"叙述者声音本来就存在"的中长篇中使用，短篇不用。

### 场景切换频率指南

| 篇幅 | 推荐切换频率 | 注意 |
|------|------------|------|
| 短篇 | 每500-800字一次场景变化 | 切换快，不留过渡段 |
| 中篇 | 每章2-3个场景 | 场景之间有清晰的过渡 |
| 长篇 | 每章1-2个场景为主 | 从容展开，场景内深挖 |

## Phase 3：交叉检查

> 生成完成后，由对应的子引擎调用 `dianya-cross-check` 执行强制质量检查。主引擎不重复定义检查规则——所有检查项、计分标准、判定阈值均在 `dianya-cross-check` 中维护。

**检查顺序**（cross-check 内部强制执行）：
1. **字数检查**（第一道门禁——内容不完整不进质量检查）
2. **AI味深度扫描**（强制加载 `dianya-novel-human`——35项指纹全景扫描 + 七铁则 + 五维评分）
3. **古风味觉** + **作家对标** + **连贯性** + **短篇/对话/武侠专项**

**判定**：任何 ❌ → 阻塞，修正后才能继续。⚠️ → 记录，下一章修正。

### 3.3 作家风格对标

将生成文本的片段与参考作家风格做对比。

**对标维度**：

| 维度 | 测量方式 | 古风相关 |
|------|---------|---------|
| 平均句长 | 总字数/句数 | 古风典雅白话通常句长偏短(15-25字) |
| 句长标准差 | 句子的长短变化幅度 | 古风要求幅度大，长短交错 |
| 对话占比 | 对话字数/总字数 | 古风小说对话更精炼 |
| 口语密度 | "了/的/吗/吧/呢/嗯/哦"等占比 | 古风减少句末"了"，代之以"已/矣"(适度) |
| 修饰比例 | 形容词+副词数量/总词数 | 古风修饰少，以名词和动词为主 |
| 动词密度 | 动作动词数量/总词数 | 越高越有古风感 |
| 段落平均句数 | 总句数/段数 | 古风段落更短，一句一段常见 |
| **古典词密度** | 古典词数/总词数 | **新增**：目标5-15% |
| **现代词密度** | 现代事物词数/总词数 | **新增**：目标0-1% |

**参考基准**（来自真实作家作品的统计，按古风浓度排列）：

| 作家 | 平均句长 | 句子节奏 | 对话占比 | 古典词密度 | 风格标签 |
|------|---------|---------|---------|-----------|---------|
| 沈从文 | 短(16-22字) | 舒缓流畅 | 中(25-35%) | 较高(~12%) | ★★★★★ 古风典雅标杆 |
| 白先勇 | 中(18-25字) | 雅致克制 | 中(25-35%) | 中(~8%) | ★★★★☆ 现代叙事·古典雅致 |
| 张爱玲 | 中长(20-30字) | 绵密精准 | 中(20-30%) | 中(~8%) | ★★★★☆ 古典比喻·现代心理 |
| 汪曾祺 | 中(20-25字) | 平淡舒缓 | 中(25-35%) | 中低(~6%) | ★★★☆☆ 口语入文·淡而有味 |
| 阿城 | 短(15-20字) | 急促干净 | 高(35-45%) | 低(~4%) | ★★☆☆☆ 极简·接近口语 |

**对标反馈**："你的文本句长分布接近[作家名]。古典词密度X%，现代词密度X%。建议[具体建议]。"

### 3.3 自定义对标

用户可以导入特定的作家作品作为对标基准。导入方式：

1. 提供目标作家的文本文件（建议3000字以上样本）
2. 引擎自动提取该作家的风格指纹（句长/节奏/用词偏好/对话模式）
3. 在交叉检查中，输出"你的文本与[作家名]的相似度分布"

### 3.4 连贯性检查（中篇·长篇强制）

> ⚠️ **强制规则**：中篇和长篇在每次阶段审计时必须执行此检查。短篇不适用。

对照追踪文档，检查以下连贯性维度：

| 检查项 | 数据来源 | 通过标准 |
|--------|---------|---------|
| 角色状态连续性 | 角色状态表 vs 正文 | 所有角色的身份/关系/位置在追踪表和正文中一致 |
| 伏笔回收进度 | 伏笔登记表 | 没有超过计划回收章节5章仍未回收的伏笔 |
| 时间线一致性 | 时间线 vs 正文 | 故事内时间推进无跳跃、无矛盾、无循环 |
| 前文引用正确性 | 章节日志 | 对话中提到的"上次"事件确实在已写章节发生过 |
| 世界观法则遵守 | 设定文档 vs 正文 | 本章没有违反已建立的世界规则 |
| 文风稳定性 | 最新3章 vs 前3章 | 句长分布、词汇密度、对话比例无突变 |
| 节奏健康度 | 最新5章 | 没有连续3章无情绪变化、没有连续2章都是慢节奏 |

**逐项判定**：
- ✅ 通过
- ⚠️ 警告（需在下一章修正，不阻塞写作）
- ❌ 不通过（**阻塞**：必须修正后才能继续）

**任何一项为❌ → 停写。** 修正追踪文档或回溯修改正文后，重新检查通过再继续。

## Phase 5：修订工作流

交叉检查发现的问题不能"再看看改改"——没有操作规范的修订要么原地打转，要么越改越烂。

### 修订分级处理

| 问题等级 | 定义 | 处理方式 | 不做什么 |
|---------|------|---------|---------|
| **轻度**（扣1-2分） | 禁用词/禁用句式零星出现 | 逐处替换。禁用词→删除或替换，禁用句式→按门禁对照表改写。 | 不要重写整段——一两个词的问题不值得。 |
| **中度**（扣3-10分） | 多处AI腔/情绪命名/结尾升华 | 标记问题段→逐段重写（保留事件，替换表达方式） | 不要全文重写。不要同时修所有问题段——一次一段，修一段检查一段。 |
| **重度**（扣10+分） | 全文AI腔/古风全无/密集现代词 | 回到一句话核心→重做大纲→重写。 | 不要在旧文本上修补——重度问题意味着方向或执行从根上错了。 |

### 修订操作顺序

不要从开篇开始改。按这个顺序：

1. **先修结尾**：结尾是全文的落点。结尾不对→全文重写。结尾对了→往前修有方向。
2. **再修转折/高潮**：这是全文最强的一段。不够强→强化。够了→以此为基准往前调整情绪。
3. **再修开篇钩子**：确认结尾和高潮之后，回头检查开篇是否指向正确方向。
4. **最后修中间段落**：逐段过门禁，替换禁用词和句式。

### 修订后的二次验证

修订完成后，重新跑 cross-check。新报告必须满足：
- 禁用词/句式清零（0处）
- 情绪命名清零（0处）
- 结尾升华清零（0处）
- 古风检查各项 ≥ -2分
- 总分为通过或轻度

**二次验证不通过→重复修订→再验证。三次修订仍不通过→回到大纲层面检查结构问题。**

## 朗读自检

古典文字讲"上口"。写完之后轻声读一遍（不出声也可以，但必须逐字过），是最朴素也最有效的自检。

读的时候注意三类不适：

| 听着不对的地方 | 可能的问题 | 处理 |
|-------------|-----------|------|
| 读着拗口，必须回头重读才通 | 句子太长或从句嵌套太深 | 拆短句。古风句子超过30字就要警觉。 |
| 读起来像翻译 | 隐藏的欧化句式——"当…时""在…中""对…来说" | 去掉欧化框架，换成直接叙述。 |
| 某段的节奏和前后不搭 | 句长突然变得均匀（3句都是20字）或段落突然变长（100+字） | 调整句长分布，恢复长短交替。 |

**朗读自检是每章写完后的最后一步**——在更新章节日志之前执行。
