# 短篇引擎（8千-3万字）

**定位：一个反转撑一篇。所有内容为唯一的情绪目标服务。**
> 先加载 `dianya-novel-engine` 获取共享风格规则（古今之辨/门禁A-D/段落六原则/三拍节奏/风格报告格式），再执行本引擎。

---

## 适用范围

| 参数 | 规格 |
|------|------|
| 字数 | 8,000-30,000 |
| 场景数 | 5-12个场景 |
| 角色数 | 2-4人（主角+关键对手+1-2配角） |
| 子线 | 无 |
| 时间跨度 | 短（数日到数月） |

## 结构：核心反转

```
开头钩子(前500字)→铺垫(3+线索)→升级(情绪加压)→反转(一节内揭示)→余韵结尾
```

| 段 | 占比 | 功能 | 必备要素 |
|----|------|------|---------|
| 开头 | 5-10% | 一句话钩住读者 | 冲突前置/反常行为/信息炸弹/代入式提问 |
| 铺垫 | 30-40% | 建立关系，埋设线索 | 至少3条反转线索，分散在不同位置 |
| 升级 | 20-30% | 冲突加剧，情绪积累 | 每节情绪递增，不允许连续2节无变化 |
| 反转 | 10-15% | 揭示真相 | 强度>前面所有节最高值，铺垫可回溯 |
| 结尾 | 5-10% | 余韵收束 | 动作/画面/短对话，不总结不感慨 |

### 四种钩子类型

**① 冲突前置**：开篇直接呈现一个正在发生的冲突。

> 示例："你走不走？"她把铜盆往地上一搁，水溅了他一脚。

**② 反常行为**：角色做出违反常识的举动，读者想知道为什么。

> 示例：他每天黄昏都去码头，站一炷香的工夫，又走回来。从来不等船。

**③ 信息炸弹**：一句蕴含巨大信息量的陈述，读者立刻被拽入情境。

> 示例：沈栀嫁进顾家的第三日，才知道夫君有个住在别院的"表妹"。

**④ 代入式提问**：用角色内心的具体困境开场，读者代入角色立场。

> 示例：要不要打开那扇门？她的手搁在铜环上，已经搁了许久。

**禁忌**：开篇避免天气描写、环境铺陈、角色外貌罗列、"很久以前"式时间交代。

### 反转强度判定

"强度 > 前面所有节最高值"的操作化定义：

| 测量维度 | 具体判定 |
|---------|---------|
| **信息冲击** | 反转揭示的信息让读者重新理解前文。可回溯——重读能发现线索。 |
| **情绪跃升** | 反转段落的情绪强度（紧张/震惊/悲恸/释然）超过铺垫段和升级段的最高点。 |
| **节奏突变** | 反转段落句长明显变短（短句加速释放），与升级段的绵密形成对比。 |
| **不可逆** | 反转之后，角色/关系/情境不可回到反转前的状态。 |

**强度不够的典型表现**：反转是读者早就猜到的；揭示方式平淡（对话交代而非场景展示）；反转后加了大量解释性段落稀释冲击力。

**简便判定法**：写完反转段落后遮住结尾段，只读开篇→铺垫→升级→反转，问自己——反转段是不是全文最强的段落？不是→重写反转。

## 写作原则

- **每句话必须有信息**：不推动剧情/不铺垫反转/不推高情绪→删
- **视角建议**：情感驱动、内心戏为主→第一人称（"我"代入感最强）；情节驱动、需要隐藏信息→第三人称限知（"她/他"，如范例）。两种均可，选定后全篇一致不混用。
- **一个反转撑一篇**：不多线、不铺世界观，所有伏笔指向同一个反转
- **节奏紧凑**：场景切换快，不留无效过渡段落
- **无连贯性追踪需求**：篇幅短，不需要伏笔登记表/角色状态表等追踪文档
- **场景字数参考**：短篇不分章，但每个场景应有实质内容。单个场景建议 800-4,000 字（中文出版计数：汉字+中文标点）——过短则事件未展开，过长则节奏拖沓。关键场景（反转段）可突破上限。全篇总字数 8,000-30,000 字，完工后检查均值。

## 短篇常见失败模式

| 失败模式 | 症状 | 原因 | 修复 |
|---------|------|------|------|
| **反转泄底** | 读到一半已猜到结局 | 线索埋得太明显，或类型套路化 | 削弱至少1条线索，其他线索改写成"双关"——有另一层合理解释 |
| **铺垫不足** | 反转像天降，情绪跟不上 | 线索不够或太隐蔽（读者无法回溯） | 增加1-2条线索，确保每条线索在反转后能被重新识别 |
| **反转不够强** | 读完了"就这？" | 反转揭示的东西对角色/读者不重要 | 问自己：这个反转改变了什么？什么都没改变→推翻重来 |
| **结尾多余** | 反转之后还有大段解释/感慨 | 不信任读者能自己理解 | 删掉所有解释性段落，只用动作/画面/短对话收束 |
| **开头浪费** | 前300字没有冲突/悬念/反常 | 从"背景介绍"起步 | 把中间最紧张的一个场景挪到开篇，用倒叙或直接开始 |
| **情绪线平** | 读起来匀速，没有起伏 | 每段情绪强度差不多 | 画出情绪曲线（1-10打分），找到平的地方加冲突 |

## 写作卡顿应急

| 卡顿类型 | 破解方法 |
|---------|---------|
| **开头写不出** | 跳过开头。从你脑子里最清晰的那个场景写起。开头最后再补。 |
| **反转想不好** | 写三版不同的反转（真相完全不同），选最意外但铺垫线索最多的那版。 |
| **情绪出不来** | 只用动作和物象写这一节，禁用所有情绪词——"怒/悲/喜/失望"全部删掉，用行为替代。 |
| **对话写僵了** | 给两个角色设定"此刻各想要什么"，重写这段对话。每句话都是角色在争取自己想要的。 |
| **写了一半不知道该怎么继续** | 回到一句话——这篇小说讲的是什么？把答案写下来，然后删掉最前面不属于核心答案的300字，从那里重启。 |

## 输出范例

以下是一段符合短篇引擎期望的文字（开篇片段）：

```
沈栀嫁进顾家的第三日，才知道夫君有个住在别院的"表妹"。

是管家说漏了嘴。他说漏嘴的时候正在给她倒茶，水柱一偏，浇在案上，他慌忙拿袖子去擦。沈栀没有追问。她只是把桌上的水渍看着，看它慢慢浸进木纹里。

第三天夜里，她问顾淮之。

"明日我去别院看看。"她的语气像在说天气。

顾淮之手里的茶盖落在碗沿上，叮的一声。他没说话。隔了很久，久到灯花爆了两次，他才说了一句："路不好走。"

"路不好走，"沈栀重复了一遍，"我走着看看。"
```

## 写后强制步骤

> ⚠️ 短篇虽然无追踪文档需求，但质量检查同样强制。以下步骤缺一不可。

**第一步：字数统计**（第一道门禁）
- 按中文出版计数法统计全文正文字数（汉字+中文标点，正则 `[一-鿿㐀-䶿　-〿＀-￯—…""]`）
- 字数 < 8,000 → **回去补写**，不进入后续检查
- 字数 > 30,000 → 考虑是否应升级为中篇

**第二步：朗读自检**
- 轻声通读全文。读到任何"卡住"的地方→标记→修正。
- 三类信号：拗口需重读→拆长句；像翻译→去欧化；节奏突兀→调句长。

**第三步：交叉校验**（调用 `dianya-cross-check`，产出书面报告。任何 ❌→阻塞）
- **字数复核**（cross-check §0）→ **AI味深度扫描**（cross-check §1：强制加载 `dianya-novel-human`，35项指纹全景扫描 + 七铁则验证 + 五维人味评分。💀致命级命中或评分<35→❌阻塞）→ **古风味觉** → **短篇专项**（禁用词密度 + 钩子力度 + 反转有效性）→ **对话专项** → **武侠专项**（若加载 dianya-wuxia）

**第四步：风格报告**（返回主引擎 Phase 4 格式）
