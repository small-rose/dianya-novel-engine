# 长篇引擎（10万字以上）

**定位：多卷本、多人物的长篇叙事。需要前期治理以保持长线不崩。**
> 先加载 `dianya-novel-engine` 获取共享风格规则，再执行本引擎。**连贯性保证机制是本引擎的硬性门禁，比中篇更严格。**

---

## 适用范围

| 参数 | 规格 |
|------|------|
| 字数 | 100,000+ |
| 章节数 | 30+章 |
| 卷数 | 2+卷（每卷10-30章） |
| 角色数 | 8+人（主角群+反派势力+功能角色） |
| 子线 | 2-3条副线 |
| 时间跨度 | 长（数年到数十年） |

## 结构：多卷弧线

```
第1卷(起点·格局建立)→第2卷(发展·冲突扩大)→第3卷(转折·危机)→第4+卷(高潮·收束)
```

| 卷 | 功能 | 字数 | 核心任务 |
|----|------|------|---------|
| 1 | 建立世界+人物 | 20-30万 | 让读者知道"这是一个什么样的故事" |
| 2 | 展开冲突 | 20-30万 | 把核心矛盾的复杂面展示出来 |
| 3-N | 转折+高潮 | 视需要 | 不断升级，不停创造新悬念 |
| 最后一卷 | 收束 | 10-20万 | 解决核心矛盾 |

## 长篇治理文件

长篇目录结构在中篇 `进度日志.md` + `大纲.md` + `正文/` + `写作物料/` 基础之上扩展。**中篇标配（根目录进度日志+大纲 + 写作物料目录中的设定/伏笔登记表/角色状态表/时间线）为所有篇幅的标配**，长篇额外增加以下目录：

```
项目/
├── 进度日志.md          # 仪表盘（每次打开项目第一眼看到）
├── 大纲.md              # 章节规划（起承转合节点 + 分章纲要）
├── 正文/               # 按章存放（与中篇结构一致）
│   ├── 第1章_章名.md
│   └── ...
├── 写作物料/            # 标配追踪文档（与中篇结构一致）
│   ├── 设定.md           # 角色档案、地名、武功、毒物、势力等设定
│   ├── 伏笔登记表.md
│   ├── 角色状态表.md
│   └── 时间线.md
├── 宪法/               # 不可改动的终极设定（长篇专属）
│   ├── 全书宪法.md
│   ├── 终极锚点.md
│   └── 世界观基石.md
├── 规划/               # 弹性规划（长篇专属）
│   ├── 卷纲.md
│   ├── 阶段规划.md
│   └── 变更日志.md
└── 设定/               # 按需翻阅，拆分子文件（长篇专属）
    ├── 角色/
    ├── 地理/
    └── 物品/
```

## 章节写作原则

- **一章一个核心事件**：不超过2个事件
- **每10-20章做阶段审计**：伏笔回收+节奏+情绪曲线
- **每卷结束做卷审计**：总结本卷得失，调整下卷规划
- **章尾制造牵引力**：悬念/未解决的张力/新信息
- **容许闲笔**：10章中可有一章"慢"，不能连续两章慢
- **多线并行**：每章紧贴一个POV，不乱切视角

### 章节标题格式（强制规则）

> ⚠️ 硬性要求：长篇每章标题必须为 **`第X章 XXXX`** 格式。标题四字（`XXXX`）为强制——不得出现非四字标题。

长篇每章标题强制格式：**`第X章 XXXX`**

其中 `XXXX` 为**四字标题**（强制四字）。长篇章数多（30+），标题的**辨识度和检索性**比中篇更重要。

| 来源 | 示例 | 适用场景 |
|------|------|---------|
| **内容四字总结** | 第1章 海棠初见 / 第15章 雨夜密谈 / 第28章 金殿对质 | 通用，最常用 |
| **典故成语化用** | 第3章 投石问路 / 第12章 暗度陈仓 / 第25章 图穷匕见 | 事件与成语契合时用 |
| **物象提取** | 第2章 银杏 / 第8章 铜镜 / 第19章 残棋 | 章节有贯穿物象时用 |
| **诗词截取** | 第5章 庭院深深 / 第18章 山雨欲来 | 意境契合时用，长篇慎用（易显酸） |

**长篇标题特殊规则**：
- **标题必须四字**——与中篇引擎同为硬性门禁。违反→修正后再继续。
- 卷首章和卷尾章建议用**内容总结**型——检索时最快定位
- 从标题应能大致判断本章属于哪一卷（卷一的标题意象与卷三应有区别）
- 标题写在正文文件的第一行：`# 第1章 海棠初见`
- **全篇标题风格必须统一**——不能在内容总结和诗词截取之间反复横跳

## 常见崩坏预防

| 崩坏类型 | 症状 | 预防 |
|---------|------|------|
| 力量失控 | 主角太强无悬念 | 设定明确的代价或限制 |
| 节奏拖沓 | 连续5章无进展 | 阶段审计发现并及时调整 |
| 伏笔断裂 | 前面埋的线后面忘了 | 维护伏笔追踪表 |
| 角色扁平 | 写多了发现角色没成长 | 每卷检查角色状态变化 |
| 烂尾危机 | 收不了场 | 写之前就确定最终锚点 |

## 连贯性保证机制（强制规则）

> ⚠️ 规则与中篇引擎一致但**更严格**——增加了世界观法则检查和卷级审计。同为硬性门禁。

### 一、必建追踪文档

中篇标配（根目录进度日志+大纲 + 写作物料目录五项） + **变更日志**（长篇额外强制）

| 文档 | 位置 | 功能 |
|------|------|------|
| 进度日志 | 根目录 | 仪表盘：章节概览 + 每章条目 |
| 大纲 | 根目录 | 章节规划：起承转合节点 + 分章纲要 |
| 设定 | 写作物料/ | 角色档案、地名、武功、毒物、势力等静态设定 |
| 伏笔登记表 | 写作物料/ | 同上 |
| 角色状态表 | 写作物料/ | 同上 |
| 时间线 | 写作物料/ | 同上 |
| **变更日志** | 规划/ | **记录每次结构性改动** |

### 二、每章写作前强制5步

与中篇引擎**完全一致**：
1. 读上章章节日志
2. 查伏笔登记表（逾期伏笔标记+处理）
3. 查角色状态表
4. 查时间线
5. 一句话确认本章意图

### 三、每章写作后强制更新 + 校验

与中篇引擎**一致**——5份追踪文档更新 + 自检 + 交叉校验。长篇自检增加两项（第6-7项），其余相同：

**第一部分：更新全部追踪文档（5份+长篇额外1份，缺一不可）**

0. **更新进度日志**（`进度日志.md`，根目录）：章节概览表追加 + 总体进度更新 + 快捷数据更新
1. **更新大纲**（`大纲.md`，根目录）：分章纲要标记✅ + 副线表/关键揭示节点更新
2. **更新伏笔登记表**（`写作物料/伏笔登记表.md`）：新增/回收/变更 + 密度统计
3. **更新角色状态表**（`写作物料/角色状态表.md`）：涉及角色全更新 + 新角色追加
4. **更新时间线**（`写作物料/时间线.md`）：追加事件行，保持时序连续
5. **更新变更日志**（`规划/变更日志.md`，长篇额外强制）：若本章有结构性改动→记录

**第二部分：自检 + 交叉校验**

6. **快速连贯性自检（8项，长篇额外两项）**：
   1. 本章角色行为与前一章状态一致？
   2. 提到的事件/物品/人物在前文有交代？
   3. 时间线连续无矛盾？
   4. 对话中的过去事件与已写章节一致？
   5. 新埋伏笔已登记？
   6. ⚠️ **本章回收的伏笔确实是在前文埋的？**（长篇伏笔线长，容易记错）
   7. ⚠️ **世界观法则未被本章事件违反？**（长篇设定多，容易崩）
   8. ⚠️ **本章字数在3,600-5,000字范围内？（超标/不足→章节日志记录原因，超标章节的下一章收紧至≤3,600字）字数按中文出版惯例：汉字+中文标点，不含空格换行和markdown格式符**

**任何一项不过→修正后再进下一章。不允许累积问题。**

7. **交叉校验**（调用 `dianya-cross-check`，产出书面报告。任何 ❌→阻塞，修正后才能进下一章。**cross-check 内部顺序：字数复核→AI味深度扫描→古风→武侠→对话→连贯性**）
   - **字数复核**（cross-check §0：标题四字检查 + 单章字数判定 + 均值趋势）→ **AI味深度扫描**（cross-check §1：强制加载 `dianya-novel-human`，35项指纹全景扫描 + 七铁则验证 + 五维人味评分。💀致命级命中或评分<35→❌阻塞）→ **古风味觉**（6项评分）→ **武侠专项**（若加载 dianya-wuxia）→ **对话专项** → **连贯性复核**

### 四、阶段审计（长篇更严格）

| 触发 | 动作 |
|------|------|
| 每写满5章 | 轻量检查：伏笔和角色状态表是否在更新 |
| 每写满10章 | 连贯性全检 + 节奏评估 |
| 每写满20章 | 卷审计：角色弧线+伏笔密度+副线交汇情况 |
| **每卷结束** | **强制卷审计**：总结本卷，调整下卷规划 |
| 结构性改动前 | 先写变更日志 + 做连贯性影响评估 |

**审计必须产出书面结果**，不得只在脑子里过。

### 五、变更管理（同中篇）

四步锁：写变更日志→评估波及范围→更新追踪文档→再动笔。

### 六、连贯性崩溃急救

分级同中篇引擎：
- 轻度→直接回溯修正
- 中度→停写，修正追踪文档后再继续
- **重度→停写，做阶段审计，未解决前禁止继续**

## 视角（POV）选择

| 视角类型 | 适用场景 | 长篇限制 |
|---------|---------|---------|
| 第三人称限知（多视角） | 多线叙事，群像，各势力博弈 | ✅ 推荐——一角色一章，章首明确标注 |
| 第三人称限知（单主角） | 主角成长线为主 | ✅ 推荐 |
| 第一人称主角 | 回忆录体/日记体/单一主角主导 | ⚠️ 可但受限——第一人称无法写"主角不在场"的场景 |
| 全知视角 | 历史演义/史诗叙事 | ⚠️ 可但危险——容易失去读者对具体角色的代入 |

**视角切换规则**：
- 每章紧贴一个POV角色，不中途切换
- 章首第一句点名"现在是谁"——用角色名或唯一身份标识
- 连续两章同一POV可以，连续三章以上同一POV建议在第三章插入一段其他角色的视角作为调剂
- 关键事件发生后，在下一个POV章节中让另一个角色"从外部"看到事件后果——形成多角度拼图

## 字数追踪与卷预算

长篇最容易失控的是字数。每写满10章做一次字数盘点。

### 每章字数标准

> 长篇每章字数标准与主引擎 `章节字数标准` 一致。基准线3,600字，常规范围3,600-5,000字。转/合阶段的关键章节可放宽至5,500字。

| 参数 | 标准 | 说明 |
|------|------|------|
| **基准线** | 3,600 字 | 长篇叙事从容，但每章仍需有核心事件 |
| **常规范围** | 3,600-5,000 字 | 低于3,600→内容不足；高于5,000→检查是否有冗余场景 |
| **溢出** | ≤ 5,500 字 | 卷尾章/大决战章允许溢出，溢出章节的下一章收紧至≤3,600字 |
| **长篇幅平均约束** | ≤ 4,000 字/章 | 长篇允许略高均值，但不应系统性地每章5,000+ |

### 字数盘点

| 盘点项 | 检查方式 | 预警线 |
|--------|---------|--------|
| 当前卷进度 | 已写字数 / 卷预算 | > 80% 时评估剩余情节是否能在预算内完成 |
| 单章膨胀 | 最近5章平均字数 vs 设定章字数 | > 150% → 章节需要拆分或精简 |
| 卷总膨胀 | 当前卷预估最终字数 / 原预算 | > 120% → 做卷结构调整（拆分卷或压缩后续情节） |

**卷预算设定**（开始每卷前确定）：

```
第N卷预算：
- 总字数目标：____ 字
- 章数：____ 章（平均每章 3,600-4,000 字）
- 本卷核心事件数：____（不超过章数）
- 允许浮动的缓冲：±10%
```

**字数超标处理**：
| 超标程度 | 处理 |
|---------|------|
| < 10% | 正常浮动，记录即可 |
| 10-20% | 检查是否有多余场景/对话，下一章收紧 |
| > 20% | 停写，做卷结构调整——拆分、压缩或重新规划 |

## 追踪文档自我校验

每次阶段审计时额外检查：

| 检查项 | 不健康信号 | 处理 |
|--------|----------|------|
| 伏笔重要度分布 | 全部"高" → 评判失效 | 重新分级：高=影响结局，中=影响本卷，低=点缀 |
| 伏笔密度 | 单章埋5+新伏笔 → 回收压力过大 | 控制每章新伏笔不超过3个 |
| 角色状态表活性 | 连续5章"无变化" → 角色停滞 | 安排状态变化事件 |
| 章节日志质量 | 连续3章摘要一句话 → 记录走形式 | 补写详细快照 |
| 变更日志频率 | 每3章都在改设定 → 前期规划不足 | 停写，重新做卷规划 |

## 写作卡顿应急

| 卡顿类型 | 破解方法 |
|---------|---------|
| **转折想不出** | 列出所有角色此刻"最不希望发生的事"，选对主角冲击最大的作为转折。 |
| **多条副线缠住了** | 画出当前的副线交汇图。每条副线一句话说明"现在在哪"。剪掉既无独立冲突又不影响主线的副线。 |
| **长篇写到中间失去方向** | 回到"全书宪法"的一句话——问自己：现在写的内容距离那句话更近了还是更远了？更远了→偏航，需要修正。 |
| **情绪出不来** | 用动作+物象重写，禁用情绪词。 |
| **角色太多写不过来了** | 检查每个角色：删除后对主线/副线有影响吗？没有→合并或边缘化。长篇不怕角色多，怕角色没有各自的功能。 |
| **卷尾写崩了** | 回到"终极锚点"——主角最终走到哪？此卷的终点离锚点更近了还是更远了？不是每卷都要前进，但每卷结束时角色状态必须有变化。 |

## 输出范例

以下是一段长篇开篇范例（第一卷第一章片段，古风典雅白话，带世界观铺垫）：

```
顾淮之第一次见到沈栀，是在他自己家里的海棠树底下。

那时候海棠正开着，粉白粉白的花瓣落了他一肩。他刚从翰林院回来，官服还没脱，就看见树下站了个姑娘——青布衫子，手里攥着一把剪刀，正踮着脚剪枯枝。她听见脚步声，转过脸来，眼睛很亮，像剪子刃上的那一道光。

"你是谁？"顾淮之问。

"沈栀。"她把剪刀收进袖子里，"你嫂子叫我来的。说这棵树有虫，不剪不行。"

顾淮之愣了一下。他嫂子三个月前就回娘家养病去了，哪里会叫人来剪树？但他没说什么。他只是点了点头，从她身边走过去，进了书房。

他关上门之后，隔着窗纸往外看。那个叫沈栀的姑娘还在剪树。她的剪刀很快，咔嚓咔嚓的，枯枝一根一根往下掉。有一根落在他大嫂平日坐的石凳上，她伸手拂掉了。

后来顾淮之才知道，沈栀根本没有见过他嫂子。

她是自己找上门来的。找上门的理由，跟那棵海棠树一样——她觉得有虫，不剪不行。

而她自己就是那把剪刀。
```
