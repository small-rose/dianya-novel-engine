# 中篇引擎（3万-10万字）

**定位：完整的起承转合，有副线，有角色深度。介于短篇锐度和长篇广度之间。**
> 先加载 `dianya-novel-engine` 获取共享风格规则，再执行本引擎。**连贯性保证机制是本引擎的硬性门禁。**

---

## 适用范围

| 参数 | 规格 |
|------|------|
| 字数 | 30,000-100,000 |
| 章节数 | 8-20章 |
| 角色数 | 4-8人（主角+2-3核心配角+反派+1-2功能角色） |
| 子线 | 1条副线 |
| 时间跨度 | 中（数月至一年） |

## 结构：完整弧线

```
起（第1-3章）→ 承（第4-8章）→ 转（第9-12章）→ 合（第13章-结尾）
```

| 阶段 | 占比 | 功能 | 每章字数 |
|------|------|------|---------|
| **起** | 15-20% | 进入故事的切口 | 3,600-5,000 |
| **承** | 35-40% | 冲突展开，副线并行 | 3,600-5,000 |
| **转** | 25-30% | 转折点，危机升级 | 3,600-5,500 |
| **合** | 15-20% | 高潮+余韵 | 3,600-5,000 |

### 章节模板

```
章首(200-500字)：钩子/快速切入前情
  ↓
中段(70%内容)：推进核心事件，牵动副线
  ↓
章尾(200-300字)：蓄力或收束，留下悬念
```

### 章节标题格式（强制规则）

> ⚠️ 硬性要求：中篇每章标题必须为 **`第X章 XXXX`** 格式。标题四字（`XXXX`）为强制——不得出现二字、三字、五字标题。

**`第X章 XXXX`**

其中 `XXXX` 为**四字标题**（强制四字），可从以下来源选取：

| 来源 | 示例 | 适用场景 |
|------|------|---------|
| **内容四字总结** | 第1章 初入顾宅 / 第5章 夜探别院 / 第9章 玉佩之谜 | 通用，最常用 |
| **典故成语化用** | 第3章 投石问路 / 第7章 杯弓蛇影 / 第12章 水落石出 | 本章核心事件恰好呼应某成语 |
| **物象提取** | 第2章 残棋 / 第6章 竹帘 / 第10章 灯花 | 本章有一个贯穿物象 |
| **诗词截取** | 第4章 秋风未起 / 第8章 更深露重 / 第11章 月满西楼 | 章节情绪与诗句意境契合 |

**拟题铁则**：
- **标题必须四字**——不得出现二字标题（如"夜路"）、三字标题（如"初见"不够四字）、五字及以上标题。违反→修正后再进下一章。这是硬性门禁。
- 标题不剧透——"第8章 沈栀坠井"不可取，改成"第8章 夜半井声"
- 标题不升华感慨——"第5章 人生若只如初见"太酸，改成"第5章 花落初见"
- 全篇标题风格统一——选定一种来源后，所有章节沿用
- 标题写在正文文件的第一行：`# 第1章 初入顾宅`

### 副线管理

- 只能有 **1条副线**，串联不超过4个章节
- 副线必须在**转折点与主线交汇**
- 副线角色不超过核心角色的一半
- 副线篇幅不超过全文的20%

### 写作原则

- **每章必须有变化**：角色状态/信息量/关系至少一个维度变化
- **章尾必须留悬念**：不需要大悬念，但要有"还想往下看"的牵引力
- **副线不喧宾夺主**
- **允许慢热**：前三章可徐徐展开

## 连贯性保证机制（强制规则）

> ⚠️ 中篇必须执行以下机制。这是硬性门禁——不建追踪文档不得写正文，不执行写前检查不得动笔，不过写后自检不得进下一章。

### 〇、项目目录结构（强制标准）

中篇小说项目必须按以下结构组织文件：

```
项目名/
├── 进度日志.md          # 仪表盘：章节概览表 + 总体进度
├── 大纲.md              # 章节规划：起承转合节点 + 分章纲要 + 副线交汇点
├── 正文/               # 所有章节文件
│   ├── 第1章_XXXX.md
│   ├── 第2章_XXXX.md
│   └── ...
└── 写作物料/            # 追踪/管理/设定文档
    ├── 设定.md           # 角色档案、地名、武功、毒物、势力等设定资料
    ├── 伏笔登记表.md    # 全部伏笔的埋设/回收状态
    ├── 角色状态表.md    # 全部角色的当前状态（动态追踪）
    └── 时间线.md        # 故事内时间推进记录
```

**铁则**：
- 进度日志和大纲置于项目根目录——打开项目第一眼看到的两个核心文件
- 正文与物料**必须分目录存放**，不得混在同一层级
- 目录名固定为 `正文` 和 `写作物料`，不得更改
- 每写完一章：更新5份追踪文档——`进度日志.md`（根目录）+ `大纲.md`（根目录）+ `写作物料/伏笔登记表.md` + `写作物料/角色状态表.md` + `写作物料/时间线.md`。5份全部更新完毕才算本章完成。
- 章节文件命名：`第X章_XXXX.md`（X为中文数字，XXXX为四字标题）
- ⚠️ 违反目录结构→修正后再继续。这是硬性门禁。
- **每章写完后必须执行交叉校验（dianya-cross-check），产出书面报告**——覆盖AI味扫描/古风味觉/武侠专项/对话专项/连贯性/字数检查。任何❌阻塞→修正后才能进下一章。

### 一、必建追踪文档（写第一章前建好）

根目录放仪表盘和规划，写作物料目录放详细追踪和设定：

| 文档 | 位置 | 功能 |
|------|------|------|
| **进度日志** | 根目录 | 仪表盘：章节概览表 + 总体进度 + 每章节条目 |
| **大纲** | 根目录 | 章节规划：起承转合节点 + 分章纲要 + 副线交汇点 |
| **设定** | 写作物料/ | 角色档案、地名、武功、毒物、势力等静态设定 |
| **伏笔登记表** | 写作物料/ | 记录每章埋的伏笔，计划回收章节 |
| **角色状态表** | 写作物料/ | 每个主角的身份/关系/心理状态（动态追踪） |
| **时间线** | 写作物料/ | 故事内时间推进 |

**伏笔登记表格式**：
```
| 伏笔ID | 埋设章节 | 内容 | 计划回收章节 | 状态 | 重要度 |
|--------|---------|------|------------|------|--------|
| F001 | 第3章 | 女主手上成色古怪的玉佩 | 第12章 | 已埋 | 高 |
```

**角色状态表格式**：
```
| 角色 | 当前身份 | 当前位置 | 关键关系状态 | 心理状态 | 上次更新章节 |
|------|---------|---------|-------------|---------|------------|
| 沈栀 | 沈家三小姐 | 京城娘家 | 与顾淮之冷战 | 迷茫但倔强 | 第8章 |
```

**进度日志每章节条目格式**（每写完一章立即追加）：
```
第N章：章名
一句话摘要：{发生了什么}
本章结束时角色状态：{主角A→状态X，主角B→状态Y}
新埋伏笔：{F00X}   已回收伏笔：{F00Y}
下章展望：{下一章要写什么}
```

### 二、每章写作前强制5步

1. **读上章章节日志**——确认角色状态和故事位置。日志不存在→先补写，不得跳过
2. **查伏笔登记表**——确认待回收伏笔即将到期；逾期5章以上的→本章必须回收或推进
3. **查角色状态表**——确认本章涉及角色当前状态。与上章矛盾→修正后再写
4. **查时间线**——确认时序连续，无跳跃矛盾
5. **一句话确认本章意图**——说不清本章要干什么→先调整大纲再动笔

### 三、每章写作后强制更新 + 校验

> ⚠️ 以下为硬性门禁——任一步骤未完成不得进下一章。

**第一部分：更新全部追踪文档（5份，缺一不可）**

0. **更新进度日志**（`进度日志.md`，根目录）：
   - 在「章节概览」表中追加本章行（章号/标题/情节摘要/字数/阶段/✅）
   - 更新「总体进度」：已完成章数、累计字数、每章均值、阶段、伏笔数量
   - 更新「快捷数据」：当前队伍/位置/待处理事项
   - 若字数超标或不足，在表中标注判定

1. **更新大纲**（`大纲.md`，根目录）：
   - 分章纲要：填入本章标题 + 情节摘要 + 标记✅
   - 副线表：若本章涉及副线交汇→更新状态
   - 关键揭示节点：若本章有揭示→标记✅

2. **更新伏笔登记表**（`写作物料/伏笔登记表.md`）：
   - 新增伏笔 → 追加行（ID/埋设章节/内容/计划回收/已埋/重要度）
   - 已回收伏笔 → 状态改为✅已回收，标注回收章节
   - 变更计划 → 更新"计划回收章节"栏
   - 更新底部「伏笔密度」统计行

3. **更新角色状态表**（`写作物料/角色状态表.md`）：
   - 凡本章涉及的角色 → 更新当前位置/关键关系/心理状态/武功/更新章
   - 本章新增角色 → 追加行
   - 状态无变化的角色 → 至少更新"更新章"字段，证明已检查
   - 已故/已离场角色 → 可保留不更新

4. **更新时间线**（`写作物料/时间线.md`）：
   - 追加本章事件行：故事时间 + 事件摘要 + 章节号
   - 注意与上一章时间连续，若有跳跃需注明原因

**第二部分：自检 + 交叉校验**

5. **快速自检**（以下7项，任何一项不过→修正后再进下一章）
   - 本章角色行为与前一章状态一致？
   - 提到的事件/物品/人物在前文有交代？
   - 时间线连续无矛盾？
   - 对话中提到的过去事件与已写章节一致？
   - 新埋伏笔已登记？
   - 世界观法则未被违反？
   - **本章字数在3,600-5,000字范围内？（第一道硬性门禁——字数不达标→回去补写，不进入交叉校验。超标需在章节日志中注明原因）字数按中文出版惯例计算：汉字+中文标点，不含空格换行和markdown格式符**

6. **交叉校验**（调用 `dianya-cross-check`，产出书面报告。任何 ❌→阻塞，修正后才能进下一章。**cross-check 内部顺序：字数复核→AI味深度扫描→古风→武侠→对话→连贯性**）
   - **字数复核**（cross-check §0：标题四字检查 + 单章字数判定 + 均值趋势）→ **AI味深度扫描**（cross-check §1：强制加载 `dianya-novel-human`，35项指纹全景扫描 + 七铁则验证 + 五维人味评分。💀致命级命中或评分<35→❌阻塞）→ **古风味觉**（6项评分）→ **武侠专项**（若加载 dianya-wuxia）→ **对话专项** → **连贯性复核**

### 四、阶段审计

| 触发 | 动作 |
|------|------|
| 每写完10章 | 连贯性全检（角色/伏笔/时间线/前文引用/文风/节奏）+ 节奏评估 |
| 结构性改动前 | 先写变更日志 + 更新所有追踪文档 |

### 五、变更管理

改设定/加角色/调情节四步锁：
1. 先写变更日志（为什么改、改了什么、影响哪些章节）
2. 评估波及范围（是否需要回溯修改已写章节）
3. 更新所有追踪文档
4. **以上三步完成才能动笔**——"悄悄改不记录"是绝对禁止的

### 六、连贯性崩溃急救

| 程度 | 症状 | 处理 |
|------|------|------|
| 轻度 | 角色名字写错/地名不一致 | 直接回溯修正 |
| 中度 | 伏笔逾期5章/角色状态矛盾 | 停写，回读相关章节，修正追踪文档后再继续 |
| 重度 | 情节走向与已写章节矛盾 | 停写，做阶段审计，重写矛盾章节或调后续大纲 |

**重度问题未解决前，禁止继续往下写新内容。**

## 视角（POV）选择

| 视角类型 | 适用场景 | 中篇限制 |
|---------|---------|---------|
| 第一人称主角 | 情感驱动，需要强烈代入感 | ✅ 推荐 |
| 第三人称限知（单视角） | 情节驱动，需要留悬念 | ✅ 推荐 |
| 第三人称限知（双视角交替） | 两条线在同一时空交织 | ⚠️ 可，每章只贴一个视角，章首明确标注 |
| 全知视角 | 信息量大、群像叙事 | ❌ 中篇不推荐——篇幅不足以支撑全知视角的深度 |

**视角切换信号**：切换视角时，新章节的第一句必须让读者立刻知道"现在是谁在经历"。用角色名或明确的身份标识开头，不用"他/她"开头。

## 追踪文档自我校验

每10章阶段审计时，额外检查追踪文档本身是否健康：

| 检查项 | 不健康信号 | 处理 |
|--------|----------|------|
| 伏笔重要度分布 | 全部标注"高" → 重要度评判失效 | 重新分级：高=影响结局，中=影响本章，低=点缀 |
| 角色状态表活性 | 连续5章写"无变化" → 角色停滞 | 给该角色安排一次状态变化事件 |
| 章节日志质量 | 连续3章摘要不超过一句话 → 记录走形式 | 补写详细状态快照 |

## 写作卡顿应急

| 卡顿类型 | 破解方法 |
|---------|---------|
| **转折想不出** | 列出所有角色此刻"最不希望发生的事"，选对主角冲击最大的那个作为转折点。 |
| **副线推不动** | 问自己：这条副线和主线在哪个点交汇？如果说不出来→副线设计有问题，重新设计交汇点。 |
| **情绪出不来** | 用动作+物象重写这一节，禁用所有情绪词。 |
| **对话写僵了** | 每个角色写下"此刻最想要什么"，重写对话。 |
| **章尾悬念枯竭** | 悬念不一定是大事——一个未打开的盒子、一句话没说全、一个角色反常的沉默，都可以。 |

## 输出范例

以下是一段中篇开篇范例（起·第一章片段，古风典雅白话）：

```
沈栀是秋天到的京城。

京城比她想的安静。马车从正阳门进去，沿街的槐树叶子黄了大半，落在青石板上，风一过就卷起来，像谁翻了旧信纸。她没有掀帘子细看。她只是从帘缝里望出去，望见城墙很高，灰扑扑的，站了一排兵。

来接她的是顾家的老管事，姓周，六十来岁，背微驼。他把马车拦在城门口，揖了一揖，说："三姑娘辛苦。"

沈栀点了点头。

周管事又说："三爷有公务，明日才回。"

沈栀又点了点头。她其实不知道三爷是谁。她只知道嫁进顾家是六月里定下的事，她爹说"门第好"，她娘没说别的，只把她那件蓝绸袄子改了三回，试到合身为止。

马车在巷子里拐了七八个弯，沈栀数着。数到第十一个的时候，她看见一扇黑漆大门，门上挂了"顾宅"两个字，字的金粉有些斑驳。

周管事推开侧门。

门后面是一个很大的院子，种了一棵银杏。树下的石桌上搁着一副残棋，棋子落了一层灰。沈栀多看了一眼——棋局下到一半，黑的围了白的一个大角。

"三姑娘里面请。"周管事的声音在院子里有点发空。

沈栀跨过门槛。

她不知道这扇门跨进去，要三年后才能再走出来。
```
